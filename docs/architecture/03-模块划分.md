# 03 模块划分

本文档将模块分为两类，避免把 PRD 功能、页面菜单和程序代码边界混用。

- **需求功能模块（Fxx）**：面向产品、权限、页面、验收，回答“系统提供什么功能、谁能用、页面如何验证”。
- **程序实现模块（Sxx/Cxx）**：按 C/S 架构拆分，面向研发架构、代码边界、接口、测试，回答“程序如何拆分、模块依赖谁、由谁持久化数据”。

编号约定：


| 前缀  | 含义                             | 主要读者                       |
| --- | ------------------------------ | -------------------------- |
| Fxx | 需求功能模块                         | 产品、测试、交付、研发                |
| Sxx | Server 端程序实现模块（USB 管控装置端服务）    | 装置端研发、后端负责人、DBA、测试架构师      |
| Cxx | Client 端程序实现模块（Windows 管理端客户端） | 前端负责人、Electron 研发、自动化测试负责人 |


---

## 1. 需求功能模块（Fxx）

需求功能模块来自 PRD，用于描述系统需要提供的业务能力、权限边界和验收规则。它不等同于程序代码模块；一个需求功能模块可以由多个 Server 端和 Client 端程序模块共同实现。

### 1.1 管理端页面型需求功能模块

#### F01 登录、登出与授权


| 项目     | 内容                                                                                                                        |
| ------ | ------------------------------------------------------------------------------------------------------------------------- |
| 功能目标   | 用户输入装置 IP、用户名、密码后连接 USB 管控装置，由装置侧完成账号鉴权、授权检查和角色返回；用户主动登出时由装置侧失效当前会话 token，管理端清空进程内会话状态并返回登录页                                                                         |
| 主要页面   | 登录页 / 独立授权页                                                                                                               |
| 权限     | 所有角色可登录；未授权时系统管理员、操作员、审计员任一有效账号鉴权通过后均可进入独立授权页；登录后的“系统管理-授权信息管理”仅系统管理员可用                                                   |
| 核心规则   | 登录页 IP 输入框为空，不预填、不记住、不展示默认 IP；账号、角色、锁定、会话均在装置侧；装置未授权时，三类有效账号均可进入独立授权页完成装置激活；授权成功后自动返回登录页并需重新登录；登录成功清零失败计数；连续 5 次失败锁定 5 分钟；主动登出需调用装置侧登出指令并失效当前会话 |
| 关联程序模块 | C01 / C02 / C03 / C06 / S08 / S09 / S10 / S11                                                                                                     |

| 功能目标   | 管理普通 U 盘白名单，支持装置端添加、管理端添加、修改权限/描述、删除             |
| 主要页面   | U盘设备控制页                                          |
| 权限     | 操作员                                              |

#### F04 策略管理

### 1.2 装置运行型需求功能模块

本节功能没有独立管理端菜单页，但属于 PRD 明确定义的核心业务能力，需在装置端运行链路、日志记录和验收测试中覆盖。

#### F08 USB 设备准入与映射


| 项目     | 内容                                                                                                           |
| ------ | ------------------------------------------------------------------------------------------------------------ |
| 功能目标   | 对普通大容量存储设备、键盘、鼠标执行准入控制和 OTG 映射                                                                               |
| 主要页面   | 无独立页面；状态和结果通过日志体现                                                                                      |
| 权限     | 装置端运行能力，不由管理端菜单权限触发                                                                                          |
| 核心规则   | 未认证 U 盘禁止映射；已认证 U 盘 / 白名单 U 盘先扫描后映射；键盘需顺序输入 1234 验证且验证过程不透传；鼠标自动映射；未知设备、不支持设备、描述符与接口能力不一致的设备默认拒绝 |
| 关联程序模块 | S01 / S02 / S03 / S04 / S05 / S07                                                                            |


#### F09 恶意代码检测


| 项目     | 内容                                               |
| ------ | ------------------------------------------------ |
| 功能目标   | 已认证 U 盘映射前由 ClamAV 全盘扫描，记录检测结果并阻断命中文件访问          |
| 主要页面   | 无独立页面；结果在恶意代码检测日志中展示                             |
| 权限     | 装置端运行能力，不由管理端菜单权限触发                              |
| 核心规则   | 命中文件添加 `[病毒禁止访问]` 前缀；病毒库不可用或扫描失败时拒绝映射；检测日志存储在装置侧 |
| 关联程序模块 | S03 / S04 / S07 / S11                            |

## 2. Server 端程序实现模块（Sxx）

### S01 USB 设备识别与映射引擎


| 项目   | 内容                                                                                     |
| ---- | -------------------------------------------------------------------------------------- |
| 职责   | 监听 USB 热插拔事件，基于 USB 描述符、接口类、接口子类、协议号和端点能力识别普通大容量存储设备、键盘和鼠标，管理设备状态机，驱动 OTG gadget 映射或禁止映射 |
| 不负责  | 白名单策略维护、病毒扫描、文件访问控制策略判断、日志查询                                                           |
| 输入   | udev 热插拔事件，USB 描述符，设备接口类型                                                              |
| 输出   | 设备状态变更事件，OTG 映射/卸载结果                                                                   |
| 依赖   | S02 / S03 / S04 / S05 / S07                                                            |
| 测试关注 | 未认证 U 盘禁止映射，未知或不支持设备默认拒绝，伪装设备不因设备名称、厂商字符串、产品字符串或白名单序列号被放行，拔出清理资源，OTG 映射失败日志记录                |


### S02 键鼠准入控制引擎


| 项目   | 内容                                           |
| ---- | -------------------------------------------- |
| 职责   | 接管键盘 HID 输入，验证 1234 顺序输入，验证通过后映射键盘；鼠标自动映射    |
| 不负责  | U 盘白名单、病毒扫描、文件访问控制                           |
| 输入   | HID 事件流，设备类型识别结果                             |
| 输出   | 键盘验证通过/失败、键盘拔出结束验证、键鼠映射结果                    |
| 依赖   | S07                                    |
| 测试关注 | 验证过程不透传受控主机，错误序列重置后继续等待，拔出释放验证状态和输入缓冲，特殊键不绕过 |


### S03 恶意代码扫描服务


| 项目   | 内容                                           |
| ---- | -------------------------------------------- |
| 职责   | 调用 ClamAV 对 U 盘执行全盘扫描，处理扫描结果，对命中文件加前缀并输出阻断标记 |
| 不负责  | 设备映射、文件访问策略配置、日志查询                           |
| 输入   | 扫描任务，U 盘挂载路径，ClamAV Unix Socket              |
| 输出   | 扫描结果、病毒名称、命中文件列表                             |
| 依赖   | ClamAV / S07                                 |
| 测试关注 | EICAR 命中，扫描中拔出，clamd 不可用，大容量扫描不阻塞 tokio 运行时  |


### S04 文件访问控制引擎


| 项目   | 内容                                                                                           |
| ---- | -------------------------------------------------------------------------------------------- |
| 职责   | 加载文件访问控制策略，基于 `/mnt/usb_raw` 构建受控文件树；真实 U 盘文件系统由 Linux 挂载层处理，目标支持 FAT32、exFAT、NTFS、ext2、ext3、ext4；通过 Rust NBD server 动态生成虚拟 exFAT 块设备视图，并对读取/写入按 5 级优先级执行阻断 |
| 不负责  | 策略导入导出、白名单维护、日志查询                                                                            |
| 输入   | 三个开关状态、38 项黑名单、4 项可执行类型、病毒文件标记、受控主机读写请求                                                      |
| 输出   | 允许内容、I/O error 阻断、自动读取阻断、整盘写保护                                                               |
| 依赖   | S05 / S07 / S11 / Linux 挂载层 / NBD                                                         |
| 测试关注 | 4 项可执行阻断，38 项黑名单大小写忽略，优先级短路，FAT32 / exFAT / NTFS / ext 系列真实介质挂载后的目录树输入，虚拟 exFAT 目录项生成，超过 4GiB 单文件的正常映射与读取，命中阻断策略时返回 I/O error                                        |


### S05 白名单策略服务


| 项目   | 内容                                                                 |
| ---- | ------------------------------------------------------------------ |
| 职责   | 管理普通 U 盘白名单增删改查；处理装置端添加与管理端添加两类白名单来源；校验序列号非空、序列号唯一、添加方式、权限和描述；为 S01 已识别出的普通大容量存储设备提供白名单准入结果和权限结果 |
| 不负责  | USB 热插拔监听、USB 描述符解析、设备类型识别、管理主机 USB 枚举、病毒扫描、文件访问控制执行、SQLite 底层读写实现 |
| 输入   | 白名单增删改查请求；装置端添加时来自 S01 的当前已连接设备识别结果；管理端添加时来自 C03/Electron 枚举的管理主机 USB 大容量存储设备信息；序列号、VID/PID、设备名称、容量、描述、权限、添加方式 |

| 测试关注 | 装置端添加仅允许当前连接且 `admission_status=addable` 的普通大容量存储设备；管理端添加写入 `add_method=management`；序列号唯一，空序列号拒绝，删除后准入失败，白名单只对已识别为普通大容量存储设备的对象生效，不放行键盘、鼠标、未知、不支持或疑似伪装设备，事务失败回滚 |

| ---- | ----------------------------------------------------- |
| 职责   | 在进程内保存当前连接状态、当前用户、角色、session token 和授权状态，并为页面提供权限展示状态 |
| 不负责  | 用户账号持久化、密码校验、角色最终授权                                   |
| 输入   | S10 登录响应，登出事件，连接断开事件                                  |
| 输出   | 当前会话内存状态，权限展示状态                                       |
| 依赖   | C03                                                   |
| 测试关注 | 不保存最近 IP，不写 localStorage 业务键，进程退出后必须重新登录              |


### C03 装置通信客户端


| 项目   | 内容                                                         |
| ---- | ---------------------------------------------------------- |
| 职责   | 与装置端 S10 建立 TCP/TLS 私有协议连接，发送登录和带 token 的业务请求，处理心跳、超时、连接断开；按消息类型分档管理请求超时（普通 15 秒 / 文件传输 300 秒），查询类指令超时自动重试 1 次；管理端添加白名单时枚举本机 USB 设备信息 |
| 不负责  | 业务规则、页面布局、装置侧权限判断                                          |
| 输入   | 装置 IP，协议请求对象                                               |
| 输出   | 协议响应，连接状态事件                                                |
| 依赖   | C02 / S10                                                  |
| 测试关注 | 连接失败提示，证书不匹配提示，管理端与装置意外断开连接不退登录页，重连后重新拉取数据                 |


### C04 页面数据装配与进程内状态


| 项目   | 内容                                                         |
| ---- | ---------------------------------------------------------- |
| 职责   | 将装置侧响应转换为页面表格、表单、分页、筛选条件和当前页面状态；管理端与装置断开连接后保留当前页面进程内状态用于展示 |
| 不负责  | 数据持久化、跨进程缓存、业务规则判断                                         |
| 输入   | C03 响应数据，页面筛选条件                                            |
| 输出   | 页面 view model，当前页面进程内状态                                    |
| 依赖   | C02 / C03                                                  |
| 测试关注 | 管理端与装置断开连接后页面不清空，查询结果不落盘，写失败不更新当前页面进程内状态                   |


### C05 文件选择与本地文件适配


| 项目   | 内容                                                     |
| ---- | ------------------------------------------------------ |
| 职责   | 通过 Electron 调用本地文件选择/保存对话框，读取升级包、病毒库包、策略文件、授权文件，保存导出文件 |
| 不负责  | 解析策略文件、校验授权文件、保存业务数据库                                  |
| 输入   | 用户选择的本地文件路径，装置侧导出字节流                                   |
| 输出   | 文件字节流，保存结果                                             |
| 依赖   | Electron 主进程 IPC / C03                                 |
| 测试关注 | 文件类型限制，取消选择处理，导出文件名，临时文件不作为业务数据缓存                      |


### C06 连接断开与错误提示处理


| 项目   | 内容                                                      |
| ---- | ------------------------------------------------------- |
| 职责   | 统一处理连接失败、意外断开连接、权限不足、未授权、写入失败、升级失败等提示；连接断开时点击下发动作给出失败提示 |
| 不负责  | 自动重试写入、本地排队、离线操作缓存                                      |
| 输入   | C03 连接事件，S10 通用错误码，页面操作上下文                              |
| 输出   | 用户可理解的错误提示和页面状态                                         |
| 依赖   | C02 / C04                                               |
| 测试关注 | 管理端与装置意外断开连接不跳登录页，下发动作失败提示，用户主动登出才跳登录页                  |
