# 06 状态机设计

---

## 1. U 盘设备状态机

每个接入的 U 盘设备维护一个独立状态机实例，由 S01 管理。

```
                  ┌──────────────────────────────────────────┐
                  │               USB 插入事件                │
                  └──────────────────┬───────────────────────┘
                                     ↓
                             ┌─────────────┐
                             │  DETECTED   │  已检测 USB 插入
                             └──────┬──────┘
                                    ↓ 描述符/接口/端点能力识别
                             ┌─────────────┐
                             │ CLASSIFYING │  接口分类
                             └──────┬──────┘
             ┌──────────────────────┼──────────────────────┐
             │ 非存储/未知/疑似伪装  │ 大容量存储接口        │
             ↓                      ↓                      │
      ┌─────────────┐              ↓ 白名单查询             │
      │  REJECTED   │              ↓                      │
      │  禁止映射    │      ┌───────┴────────┐             │
      └─────────────┘      │ 不在白名单       │ 在白名单     │
             │             ↓                 ↓             │
             │      ┌─────────────┐   ┌─────────────┐      │
             │      │  BLOCKED    │   │  SCANNING   │      │
             │      │  禁止映射    │   │  扫描中     │      │
             │      └─────────────┘   └──────┬──────┘      │
                                     │                      │
                  ┌──────────────────┼──────────────┐       │
                  │ 扫描完成          │ 扫描失败      │ 发现病毒
                  ↓                  ↓              ↓
           ┌──────────┐       ┌──────────┐  ┌──────────────┐
           │  CLEAN   │       │  SCAN_   │  │  INFECTED    │
           │  干净     │       │  FAILED  │  │  检出病毒     │
           └────┬─────┘       └──────────┘  └──────┬───────┘
                ↓ NBD 受控映射                         ↓ 命中文件重命名
                ↓ OTG 映射                            ↓ 继续 NBD 受控映射
           ┌────────────┐                            ↓ OTG 映射
           │   MAPPED   │◄───────────────────────────┘
           │  已映射     │
           └────┬───────┘
                │
                ↓ USB 拔出事件
           ┌────────────┐
           │  REMOVED   │
           │  已拔出     │
           └────────────┘
```

**状态转换表：**

| 当前状态 | 事件 | 下一状态 | 动作 |
|---|---|---|---|
| DETECTED | USB 插入 | CLASSIFYING | 读取设备描述符、配置描述符、接口类、接口子类、协议号和端点能力 |
| CLASSIFYING | 非存储接口、未知接口、描述符与能力不一致或疑似伪装 | REJECTED | 默认拒绝映射，记录 USB 审计日志 |
| CLASSIFYING | 识别为大容量存储接口 | BLOCKED / SCANNING | 进入白名单查询 |
| CLASSIFYING | 白名单查询：不在白名单 | BLOCKED | 记录 USB 审计日志（未认证设备） |
| CLASSIFYING | 白名单查询：在白名单 | SCANNING | 提交病毒扫描任务（S03） |
| SCANNING | 扫描完成：未发现病毒 | CLEAN | 记录恶意代码检测日志（未发现）|
| SCANNING | 扫描完成：发现病毒 | INFECTED | 命中文件重命名，记录恶意代码检测日志 |
| SCANNING | 扫描失败/病毒库不可用 | SCAN_FAILED | 记录恶意代码检测日志（扫描失败） |
| CLEAN | NBD 受控块设备启动 + OTG 映射成功 | MAPPED | 记录 USB 审计日志（映射成功） |
| INFECTED | NBD 受控块设备启动 + OTG 映射成功 | MAPPED | 记录 USB 审计日志（含病毒，映射成功）|
| REJECTED | USB 拔出 | REMOVED | 记录 USB 审计日志（拔出） |
| BLOCKED | USB 拔出 | REMOVED | 记录 USB 审计日志（拔出） |
| SCAN_FAILED | USB 拔出 | REMOVED | 记录 USB 审计日志（拔出） |
| MAPPED | USB 拔出 | REMOVED | 卸载 `/mnt/usb_raw`，停止 NBD server，清理 OTG 映射，记录 USB 审计日志（拔出） |
| 任意状态 | 异常（内核错误等） | REMOVED | 清理资源，记录错误日志 |

**注意：**
- REJECTED、BLOCKED 和 SCAN_FAILED 状态下设备不映射到受控主机
- 白名单只放行已识别为普通大容量存储设备的对象，不因设备名称、厂商字符串、产品字符串或序列号相似而放行键盘、鼠标、未知或不支持设备
- INFECTED 状态下允许映射，但命中病毒的文件在 NBD 层读取返回 I/O error，不向受控主机暴露真实内容

## 2. 键盘设备状态机

```
                   ┌──────────────────┐
                   │   USB 插入事件    │
                   └────────┬─────────┘
                            ↓
                   ┌─────────────────┐
                   │   KB_DETECTED   │  已检测为键盘
                   └────────┬────────┘
                            ↓ 接管 HID，阻断透传
                   ┌─────────────────┐
                   │  KB_WAITING     │  等待用户输入（无固定超时）
                   │  等待验证       │
                   └────────┬────────┘
                            │
              ┌─────────────┼──────────────┬──────────────┐
              │ 顺序输入 1234 │ 输入错误      │ USB 拔出      │
              ↓             ↓              ↓              │
       ┌─────────────┐ ┌──────────────────────┐ ┌─────────────┐
       │  KB_MAPPED  │ │      KB_WAITING       │ │  KB_REMOVED │
       │  键盘已映射  │ │ 清空输入序列，继续等待 │ │  验证结束    │
       └──────┬──────┘ └──────────────────────┘ └─────────────┘
              │
              ↓ USB 拔出
       ┌─────────────┐
       │  KB_REMOVED │
       └─────────────┘
```

**说明：**
- KB_WAITING 状态下，键盘的 HID 输入由装置侧捕获，不透传到受控主机
- 1234 必须是顺序输入，中途错误需清空当前输入序列，并继续等待下一次完整输入
- 键盘验证不设置固定等待超时；键盘拔出时结束本次验证流程，释放 HID 接管状态和临时输入缓冲

## 3. 管理端连接状态机

```
     ┌───────────────────────┐
     │   初始状态（未连接）     │
     └──────────┬────────────┘
                ↓ 用户点击登录
     ┌──────────────────────┐
     │   CONNECTING         │
     │   连接中（TCP/TLS）   │
     └──────────┬───────────┘
                │
     ┌──────────┼──────────────┐
     │ 连接失败   │ 连接成功     │
     ↓          ↓              │
  DISCONNECTED  │              │
  未连接         │              │
  （提示错误）   ↓              │
             AUTHENTICATING    │
             装置侧账号鉴权     │
                    ┌─────────┼──────────────┐
                    │ 鉴权失败  │ 鉴权成功      │
                    ↓          ↓              │
                 DISCONNECTED CHECK_LICENSE  │
                 未连接        检查授权        │
                              ┌┴────────────┐
                              │             │
                       未授权         已授权
                              ↓             ↓
                       AUTH_REQUIRED   LOADING_CONFIG
                       独立授权页        拉取配置中
                                            ↓ 配置拉取完成
                         ┌────────────┐
                         │ CONNECTED  │
                         │  已连接     │
                         └─────┬──────┘
                               │
                ┌──────────────┼─────────────┐
                │ 网络异常/超时  │ 用户主动登出  │
                ↓              ↓
           DISCONNECTED    DISCONNECTED
           连接断开          已登出（跳登录页）
           保留当前页面进程内状态
```

装置未授权时，任一有效账号鉴权通过后均进入 `AUTH_REQUIRED`。独立授权页属于进入主系统前的装置激活流程，不按登录后三权权限限制；授权成功后返回登录页并清空当前临时会话，用户需重新登录。

**全局状态字段（Pinia store）：**

| 字段 | 类型 | 说明 |
|---|---|---|
| `connectionStatus` | enum | DISCONNECTED / CONNECTING / CONNECTED |
| `deviceIP` | string | 当前进程内输入并连接的装置 IP，不持久化 |
| `currentUser` | object | `{username, role}` |
| `sessionToken` | string | 装置端返回的临时会话 token，仅进程内保存 |
| `deviceConfig` | object | C04 装配的当前页面状态，仅进程内保存，管理端与装置断开连接后保留用于展示 |

**管理端与装置断开连接行为：**

- 管理端与装置意外断开连接不自动跳登录页，不清空当前页面展示内容。
- 任何需要下发到装置侧的动作在点击时失败提示。
- 管理端关闭、崩溃或用户主动登出后，进程内状态清空，重新打开必须重新输入 IP、用户名和密码。

## 4. 病毒扫描任务状态机

```
   提交扫描任务
         ↓
   ┌──────────────┐
   │   PENDING    │
   │   等待执行    │
   └──────┬───────┘
          ↓ 任务调度
   ┌──────────────┐
   │   RUNNING    │
   │   扫描中      │
   └──────┬───────┘
          │
    ┌─────┼──────────────┐
    │     │              │
    ↓     ↓              ↓
  扫描完成  USB 拔出     clamd 异常
  ┌──────┐  ┌──────────┐  ┌──────────┐
  │CLEAN │  │CANCELLED │  │ FAILED   │
  │清洁  │  │已取消    │  │失败      │
  └──────┘  └──────────┘  └──────────┘
  或
  ┌──────────┐
  │INFECTED  │
  │发现病毒  │
  └──────────┘
```

## 5. 系统升级状态机

```
初始状态（正常运行）
    ↓ 收到升级包
┌──────────────┐
│  VALIDATING  │
│  校验中       │
└──────┬───────┘
       │
  ┌────┼────────────┐
  │ 校验失败    校验成功│
  ↓              ↓
IDLE          UPGRADING
恢复正常        升级中（替换程序）
              ↓
         ┌───┴─────────┐
         │ 升级成功   失败│
         ↓             ↓
      RESTARTING    ROLLBACK
      重启服务        回滚（恢复原程序）
         ↓             ↓
      IDLE（新版本）  IDLE（原版本）
```
