# 10 自动化测试设计

---

## 说明

测试策略从架构阶段确立，确保代码写完就有对应测试。测试分四层：单元测试、集成测试、接口测试、E2E 测试。

---

## 1. 测试层次与职责

| 层次 | 工具 | 覆盖范围 | 何时运行 |
|---|---|---|---|
| 单元测试 | Rust `cargo test` + `tokio::test` + `mockall`（装置端）/ Vitest + Vue Test Utils（管理端）| 模块内函数、业务逻辑、边界条件 | 每次代码提交 |
| 集成测试 | Rust 集成测试（装置端）| 多模块协作、数据库读写、ClamAV mock | 每次代码提交 |
| 协议接口测试 | Rust 编写的协议测试客户端 | 私有协议消息的请求/响应 | 每次代码提交 |
| E2E 测试 | Playwright（Electron）| 完整用户操作流程 | PR 合并前 |

## 2. 装置端单元测试用例

### S01 USB 设备识别与映射引擎

| 测试场景 | 预期结果 |
|---|---|
| 插入不在白名单的 U 盘 | 状态变为 BLOCKED，记录未认证日志 |
| 插入在白名单（只读权限）的 U 盘 | 状态变为 SCANNING，触发扫描任务 |
| 扫描未发现病毒，NBD 受控块设备启动成功 | 状态变为 MAPPED，记录映射成功日志 |
| 扫描发现病毒，S04 构建受控文件树 | 状态变为 MAPPED（含病毒），命中文件在虚拟 exFAT 目录项添加 [病毒禁止访问] 前缀，读取返回 I/O error |
| 伪装设备：设备名称像 U 盘但接口类为 HID | 不进入 U 盘白名单放行流程，按 HID/未知接口规则处理 |
| 伪装设备：序列号命中白名单但接口能力不是大容量存储 | 不映射为 U 盘，记录伪装或接口不匹配日志 |
| 未知或不支持设备接入 | 默认禁止映射，记录 USB 审计日志 |
| 描述符与端点能力不一致 | 默认拒绝映射，记录 USB 审计日志 |
| U 盘拔出（MAPPED 状态）| `/mnt/usb_raw` 卸载，NBD server 停止，OTG 映射解除，记录拔出日志 |

### S02 键鼠准入控制引擎

| 测试场景 | 预期结果 |
|---|---|
| 键盘插入，顺序输入 1-2-3-4 | 状态变为 KB_MAPPED，键盘映射到受控主机 |

| 鼠标映射失败 | 状态变为 MOUSE_REJECTED，记录鼠标映射失败日志 |

### S09 授权与系统维护服务

| 测试场景 | 预期结果 |
|---|---|
| 获取机器码 | 返回 Base64 编码的机器码和二维码 PNG |

| 系统升级包版本低于或等于当前版本 | 返回 VERSION_TOO_LOW (0x0601)，不升级 |

| 修改设备描述（合法：字母/数字/下划线，≤32 字符）| 修改成功，记录操作日志 |
| 修改设备描述（非法字符或超长）| 返回 VALIDATION_FAILED (0x0005) |

### S10 私有协议 API 网关

| 测试场景 | 预期结果 |
|---|---|
| 合法帧头（Magic + 消息类型 + SeqID + PayloadLength）| 正确解析并路由到对应处理模块 |
| Magic Number 不匹配（非 0x55534243）| 拒绝帧，断开连接 |
| PayloadLength 超过 128MB 上限 | 拒绝帧，返回错误 |
| PayloadLength 与实际载荷长度不一致 | 拒绝帧，返回错误 |
| 未知消息类型码（不在已定义区间内）| 返回 RSP_COMMON 错误 |
| 无 session_token 的业务请求（非登录/心跳）| 返回 UNAUTHENTICATED (0x0001) |
| session_token 过期（超过 5 分钟无业务请求）| 返回 UNAUTHENTICATED (0x0001) |
| session_token 有效，角色无当前接口权限 | 返回 PERMISSION_DENIED (0x0003) |
| session_token 有效，角色有权限 | 请求路由到目标模块并返回正确响应 |
| 已有一个管理端连接时，第二个管理端尝试连接 | 拒绝第二个连接 |
| Protobuf 载荷反序列化失败（格式损坏）| 返回 VALIDATION_FAILED (0x0005) |
| 并发请求（同一连接多条业务请求）| 按 SequenceID 正确匹配请求和响应 |
| 心跳请求（CMD_HEARTBEAT）| 立即回复 RSP_HEARTBEAT，不更新会话超时计时器 |
| 连续 3 次心跳无响应 | 装置端标记连接断开，释放会话资源 |

## 5. 管理端 E2E 测试

使用 Playwright + Electron 支持，针对完整应用执行。

### 登录流程

```
1. 启动 Electron 应用
2. 输入正确装置 IP（指向 mock 装置服务）
3. 输入正确用户名和密码
4. 验证：进入对应角色的默认页面（admin→用户管理，operator→文件访问控制，auditor→日志管理）
5. 验证：菜单只展示该角色有权限的菜单项
6. 点击登出
7. 验证：返回登录页，进程内会话状态清空
```

### 未授权装置激活流程

```
1. mock 装置服务返回 authorized=false
2. 分别使用 admin / operator / auditor 三类有效账号登录
3. 验证：三类账号均跳转独立授权页
4. 点击下载机器码，验证展示机器码文本、二维码和二维码下载按钮
5. 上传有效 .txt 授权文件
6. 验证：授权成功后自动返回登录页，不能直接进入主系统
7. 再次输入装置 IP、用户名、密码登录
8. 验证：按角色进入对应默认页面（admin→用户管理，operator→文件访问控制，auditor→日志管理）
```

### 操作员完整流程

```
1. operator 登录
2. 进入文件访问控制页
3. 勾选"可执行程序访问控制"开关
4. 添加黑名单后缀 ".test"
5. 验证：页面展示新条目，开关状态正确
6. 删除黑名单后缀（二次确认弹窗）
7. 验证：条目消失
```

### 管理端与装置断开连接流程

```
1. operator 登录成功
2. mock 服务模拟连接断开
3. 验证：仍停留在当前页面，不跳回登录页，当前页面进程内状态保留
4. 点击策略修改按钮
5. 验证：提示"装置已断开，请重新连接后再操作"
6. mock 服务恢复连接
7. 重新登录
8. 验证：重新拉取装置侧数据
```

### 三权分立验证

```
1. 分别以 admin / operator / auditor 登录
2. 验证每个角色只能看到规定的菜单
3. 通过路由直接访问无权限路径，验证被拒绝（重定向登录页）
```

### Windows 简体/繁体中文环境兼容

```
1. mock Electron 文件对话框返回包含简体中文的路径，例如 C:\用户\测试账号\桌面\策略文件.bin
2. mock Electron 文件对话框返回包含繁体中文的路径，例如 C:\使用者\測試帳號\桌面\授權文件.txt
3. 执行策略导入、策略导出、授权文件上传、系统升级包选择、病毒库升级包选择、日志导出
4. 验证：路径传递、文件读取、文件保存均成功，不出现乱码、截断或路径解析失败
5. mock 装置返回中文日志内容、中文设备描述、中文错误信息
6. 验证：页面展示内容不乱码，导出的日志内容为 UTF-8
7. 验证：日志导出 .zip 内文件名使用 UTF-8 编码，在 Windows 简体/繁体环境下解压后文件名可读
```
