# 01 系统架构总览

## 1. 业务目标理解

USB 安全管理系统在受控主机与 USB 外设之间插入一个硬件中间件（RK3568 装置），实现：

- **准入控制**：未认证 U 盘禁止映射到受控主机
- **病毒扫描**：已认证 U 盘映射前全盘扫描，命中文件标记并禁止访问
- **文件访问控制**：按策略阻断特定类型文件的访问
- **键盘/鼠标安全映射**：键盘需验证，鼠标自动映射
- **审计日志**：全程记录设备接入、映射、文件访问和管理操作
- **管理控制**：管理人员通过 Windows 客户端程序配置策略、管理授权、审计日志

核心特征：**不改变受控主机软件环境**，管控逻辑完全在装置侧执行。

---

## 2. 整体架构

系统采用 C/S 架构，由管理端主动连接 USB 管控装置完成配置、审计和系统维护。USB 管控装置作为服务端部署在 RK3568 硬件上，负责所有安全判断、策略执行、数据存储和 USB 映射；管理端作为客户端部署在 Windows 管理主机上，仅负责页面交互、文件选择和结果展示；受控主机不属于 C/S 通信双方，只使用装置通过 OTG USB 映射出的 U 盘、键盘和鼠标。

```
USB 外设
   │ USB 接入
   ▼
┌──────────────────────────────────────────────────────────────┐
│ USB 管控装置（服务端 / RK3568 / Ubuntu 22.04）                  │
│ - 执行设备识别、准入判断、病毒扫描、文件访问控制和 OTG 映射           │
│ - 执行登录鉴权、角色权限校验、策略导入导出、授权和系统维护            │
│ - 持有账号、授权、白名单、文件访问控制策略和三类业务日志              │
│ - 持有病毒库、升级包接收文件、授权文件资源、安装配置资源和             │
│   Rust 服务运行日志                                            │
└───────────────┬───────────────────────────────▲──────────────┘
                │ OTG USB 映射                   │ TCP/TLS 私有协议
                ▼                                │ 管理端主动连接装置
┌──────────────────────────────┐     ┌───────────┴──────────────────┐
│ 受控主机                      │     │ 管理主机（客户端 / Windows）     │
│ - Windows / Linux            │     │ - Electron + Vue 3 管理控制程序 │
│ - 无需安装任何软件              │     │ - 页面交互、文件选择、结果提示    │
│ - 使用映射后的 U盘/键盘/鼠标     │     │ - 仅保存当前进程内状态           │
└──────────────────────────────┘     │ - 不持久化业务/账号/日志/IP      │
                                      └──────────────────────────────┘
```


| 物理节点     | C/S 角色   | 部署环境                                | 职责边界                                                         |
| -------- | -------- | ----------------------------------- | ------------------------------------------------------------ |
| USB 管控装置 | 服务端      | RK3568 / Ubuntu 22.04 / kernel 4.19 | 执行 USB 设备识别、准入判断、病毒扫描、文件访问控制、OTG 映射、用户鉴权、权限校验、策略管理、日志记录和系统维护 |
| 管理主机     | 客户端      | Windows 10 / Windows 11             | 运行 Electron + Vue 管理控制程序，主动连接装置，提供页面交互、文件选择、结果提示和当前进程内状态管理   |
| 受控主机     | 非 C/S 节点 | Windows / Linux                     | 无需安装软件，仅使用 USB 管控装置通过 OTG USB 映射后的 U 盘、键盘和鼠标                 |


**数据与日志边界：**

1. 账号、授权、白名单、文件访问控制策略、USB 审计日志、恶意代码检测日志和操作日志均以装置侧 SQLite 为权威数据源。
2. 装置侧本地文件用于保存病毒库、升级包接收文件、授权文件资源、安装配置资源和 Rust 服务运行日志。
3. Rust 服务运行日志属于装置端程序排查日志，随装置端服务记录在本地文件中，可包含 `trace`、`debug`、`info`、`warn`、`error` 等级。
4. Rust 服务运行日志不属于 USB 审计日志、恶意代码检测日志、操作日志，不在管理端日志管理页面展示、查询、导出或清理。
5. 管理端不落盘保存任何业务数据、账号数据、日志数据、最近连接 IP、当前页面进程内状态或查询结果。

## 3. 分层说明

### 装置端分层


| 层次          | 职责                                                                                                                                             |
| ----------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| USB 接入与准入层  | 监听 USB 热插拔事件，基于 udev、USB 描述符、接口类、接口子类、协议号和端点能力识别普通 U 盘、键盘、鼠标等设备类型；按 PRD 定义分别执行 U 盘白名单准入、键盘顺序验证、鼠标自动放行；不支持或无法识别的设备默认禁止映射                     |
| OTG 映射层      | 通过 USB gadget configfs 将受控 U 盘（f_mass_storage）、键盘和鼠标（HID gadget）映射到受控主机，管理映射建立、卸载和资源清理的生命周期                     |
| 恶意代码扫描层     | 调用 ClamAV（clamd）执行 U 盘全盘扫描，处理扫描结果，对命中文件添加 `[病毒禁止访问]` 前缀并输出病毒文件标记；病毒文件访问阻断由文件访问控制层执行                                                            |
| 文件访问控制层     | 基于 Linux 挂载层读取真实 U 盘文件系统（目标支持 FAT32、exFAT、NTFS、ext2、ext3、ext4，实际支持范围以装置端系统镜像启用的内核驱动和挂载工具为准），通过 Rust NBD 生成受控虚拟 exFAT 块设备视图，按病毒文件、可执行程序控制、文件类型黑名单、介质自动读取控制、读写权限的优先级执行访问判断，并在块设备响应层返回 I/O error 实现阻断；受控主机不直接访问真实 U 盘文件系统                           |
| 配置与数据管理层    | 持久化账号、授权、白名单、文件访问控制策略、设备描述、版本信息等装置侧权威数据，提供事务、迁移、掉电恢复和策略加载能力                                                                                    |
| 业务日志层       | 记录、查询、导出、清理 USB 审计日志、恶意代码检测日志和操作日志，执行装置整体存储超过 80% 后按日志类别滚动覆盖规则；不处理 Rust 服务运行日志                                                                 |
| Rust 服务运行日志 | 记录装置端程序运行、调试和异常排查日志，支持 `trace`、`debug`、`info`、`warn`、`error` 等级；日志以本地文件方式保存，不在管理端日志管理页面展示、查询、导出或清理；当装置整体存储使用率超过 80% 时，按运行日志自身滚动策略清理或覆盖最旧运行日志文件 |
| 系统维护层       | 处理系统升级、病毒库升级、授权文件导入、机器码生成、设备描述修改和服务健康检查等维护能力                                                                                                   |
| 管理协议层       | 监听 TCP/TLS 私有协议连接，解析协议帧，处理登录鉴权、会话 token、授权状态、角色权限校验和请求路由                                                                                       |


### 管理端分层


| 层次       | 职责                                                                        |
| -------- | ------------------------------------------------------------------------- |
| UI 交互层   | Electron + Vue 3 实现桌面 GUI，按装置侧返回的角色权限渲染菜单、页面、按钮和提示                        |
| 进程内状态层   | 仅在内存保存当前连接、当前会话、当前页面状态和当前请求结果；进程退出后全部清空                                   |
| 管理协议客户端层 | 建立和维护与装置端的 TCP/TLS 私有协议连接，TLS 握手后校验证书指纹（与安装包内置值比对，不一致则拒绝连接并提示"版本不兼容，请升级管理端"），发送带会话 token 的业务指令，处理超时、连接断开和重连后的数据刷新            |
| 本地文件适配层  | 调用 Windows 文件选择和保存能力，读取用户选择的授权文件、策略文件、升级包、病毒库升级包，并保存装置侧导出的策略或日志文件；不缓存业务数据 |
| 持久化边界    | 管理端不落盘保存业务数据、账号数据、日志数据、最近连接 IP、当前页面进程内状态或查询结果                             |

## 4. 关键设计决策

### 4.1 单体模块化，不用微服务

装置侧为嵌入式 Ubuntu 环境（4GB 内存/32GB 存储），管理端为一对一单用户桌面程序。无需微服务，单体分模块结构即可满足需求，且便于模块边界划分、测试覆盖和后续维护。

### 4.2 配置主权在装置侧

白名单、文件访问控制策略、设备描述、授权、用户账号、会话状态和三类日志均以装置侧为权威数据源。管理端每次登录/重连后从装置侧读取当前页面所需数据；管理端与装置断开连接后保留当前页面进程内状态，用户点击下发类动作时提示失败，重连后重新读取装置侧数据。

### 4.3 管理端零业务持久化

管理端不得持久化任何业务数据、账号数据或日志数据：

- 不保存用户账号、密码 hash、角色、锁定状态或会话 token 到磁盘
- 不保存 USB 审计日志、恶意代码检测日志、操作日志、运行日志或异常日志到磁盘
- 不保存最近连接 IP、查询条件、当前页面进程内状态、策略副本或导出中间文件
- 关闭、崩溃、重启后必须重新输入装置 IP、用户名和密码

管理端连接后从装置侧拉取三类日志展示，无需集中日志系统。

### 4.4 不做策略暂存和增量同步

策略导入采用整体覆盖，失败原策略不变。不做增量合并和暂存，降低复杂度和出错概率。

### 4.5 USB 设备安全准入默认拒绝

USB 管控装置必须按 udev、USB 描述符、接口类、接口子类、协议号和端点能力识别设备，不得仅按设备名称、厂商字符串、产品字符串或白名单序列号放行。普通 U 盘、键盘、鼠标按 PRD 定义分别进入对应准入和映射流程；不支持或无法识别的设备默认禁止映射并记录 USB 审计日志。

白名单只作用于识别为普通 U 盘的设备，不作为键盘、鼠标或其它不支持设备的放行依据。

## 5. 数据流向总览

```
管理端操作 ──私有协议──→ 装置端 API ──→ 鉴权/权限校验 ──→ 配置管理层 ──→ SQLite
                                    ↓
                              USB设备管理层 ←── USB热插拔事件
                                    ↓
                              病毒扫描层（ClamAV）
                                    ↓
                              文件访问控制层
                                    ↓
                              OTG 映射 ──→ 受控主机可见
                                    ↓
                              日志层（USB/恶意代码/操作） ──→ SQLite

管理端查询 ──私有协议──→ 装置端 API ──→ 日志层 ──→ 返回日志数据
```

## 6. 非本版本能力边界

以下内容用于明确本版本不实现范围，避免研发按隐藏需求处理。


| 能力         | 本版本处理                           | 架构边界                                     |
| ---------- | ------------------------------- | ---------------------------------------- |
| 安全 U 盘专属能力 | 不支持安全 U 盘芯片能力，安全 U 盘按普通 U 盘流程处理 | 设备识别仅按本版本定义的 USB 描述符、接口类、接口子类、协议号和端点能力执行 |
| 键盘密钥配置     | 固定验证序列为 `1234`，不提供配置入口          | 键盘验证仅执行完整顺序匹配，不提供密钥管理页面和接口               |
| 集中管理       | 不支持集中平台或多装置统一管理                 | 管理端与单台装置直连，装置侧为业务数据权威源                   |
| 在线升级       | 不支持在线自动更新                       | 程序升级和病毒库升级均通过管理端上传本地升级包执行                |
| 指示灯联动      | 不支持指示灯业务联动                      | 本版本不定义 GPIO 控制接口和页面交互                    |
