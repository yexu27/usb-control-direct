# 统一结果码表

装置端通过 RspCommon.result_code 或特定响应消息的 result_code 字段返回数字结果码。管理端根据结果码决定提示方式。

成功/失败契约：

- 成功：`success = true`，`result_code = 0x0000`，`error_message` 为空字符串。
- 失败：`success = false`，`result_code` 为本表定义的非零值，`error_message` 为装置端填写的中文提示文本。

结果码编号与消息类型码编号是独立命名空间（结果码在 Protobuf 载荷内，消息类型码在协议帧头），不会产生歧义。

---

## 1. 编码规则

结果码为 16 位无符号整数（proto 中使用 `int32` 传输），按模块分段：

| 段 | 模块 | 范围 |
|---|---|---|
| 0x0000 | 成功 | 仅 0x0000 |
| 0x00xx | 全局错误 | 0x0001 - 0x000F |
| 0x01xx | 登录鉴权 | 0x0101 - 0x01FF |
| 0x02xx | 白名单管理 | 0x0201 - 0x02FF |
| 0x03xx | 文件访问控制策略 | 0x0301 - 0x03FF |
| 0x04xx | 策略导入导出 | 0x0401 - 0x04FF |
| 0x05xx | 日志管理 | 0x0501 - 0x05FF |
| 0x06xx | 系统管理 | 0x0601 - 0x06FF |
| 0x07xx | 用户管理 | 0x0701 - 0x07FF |

判断规则：
- `result_code == 0x0000` → 成功。
- `result_code != 0x0000` → 失败，按本表匹配提示文案；未匹配到已知结果码时，直接展示 error_message 原文。
- 全局结果码（0x0001 - 0x0007）优先级高于模块结果码：装置端先检查会话、授权、权限等全局条件，通过后再执行业务逻辑。

---

## 2. 成功码

| 结果码 | 常量名 | 含义 | 管理端处理 |
|---|---|---|---|
| 0x0000 | SUCCESS | 操作成功 | 按业务逻辑处理响应数据 |

---

## 3. 全局结果码

全局结果码适用于所有消息类型，任何接口均可能返回。

| 结果码 | 常量名 | 含义 | 管理端提示文案 |
|---|---|---|---|
| 0x0001 | UNAUTHENTICATED | 会话无效或已过期 | 清空进程内会话状态，跳转登录页（无弹窗提示） |
| 0x0002 | UNAUTHORIZED | 装置未授权 | "装置未授权，请先完成授权" |
| 0x0003 | PERMISSION_DENIED | 当前角色无权限 | "当前用户无权执行此操作" |
| 0x0004 | ACCOUNT_LOCKED | 账号已锁定 | "用户已被锁定，请5分钟后重试" |
| 0x0005 | VALIDATION_FAILED | 请求参数校验失败 | 展示 error_message 原文 |
| 0x0006 | DEVICE_BUSY | 装置正在执行其他操作 | "装置正在处理其他操作，请稍后重试" |
| 0x0007 | INTERNAL_ERROR | 装置内部错误 | "装置内部错误，请联系管理员" |

## 4. 登录鉴权结果码（0x01xx）

登录结果码通过 RspLogin.result_code（0x0002）返回。授权文件上传结果码通过 RspUploadLicense.result_code（0x0008）返回。

| 结果码 | 常量名 | 适用消息码 | 含义 | 管理端提示文案 |
|---|---|---|---|---|
| 0x0101 | USER_OR_PASSWORD_ERROR | 0x0001 | 用户名或密码错误 | "用户名或密码错误" |
| 0x0102 | DEVICE_UNAUTHORIZED | 0x0001 | 装置未授权，需进入授权页 | 跳转独立授权页 |
| 0x0103 | LICENSE_FORMAT_ERROR | 0x0007 | 授权文件格式错误 | "授权文件格式错误，请检查文件" |
| 0x0104 | LICENSE_VERIFY_FAILED | 0x0007 | 授权文件校验失败（签名不匹配、与装置不匹配或已过有效期） | "授权文件校验失败，请确认文件与本装置匹配" |
| 0x0105 | LICENSE_EXPIRED | 0x0007 | 上传的授权文件已过有效期 | "授权文件已过有效期，请重新获取授权文件" |

说明：

- RspLogin.result_code 可返回全局结果码 ACCOUNT_LOCKED（0x0004，连续 5 次登录失败锁定）。
- CMD_AUTH_STATUS_QUERY（0x0003）的授权状态通过 RspAuthStatus.auth_status 字段返回（authorized / unauthorized / expired / failed），不使用结果码机制。管理端根据 auth_status 值判断授权状态并展示对应提示。

## 11. 结果码使用规则

- 装置端在 RspCommon.result_code 或特定响应的 result_code 字段中返回上述数字值。
- 管理端优先按 result_code 数字值匹配提示文案；未匹配到已知结果码时，直接展示 error_message 原文。
- 全局结果码（第 3 节）优先级高于模块结果码：装置端先检查会话、授权、权限等全局条件，通过后再执行业务逻辑。
- UNAUTHENTICATED（0x0001）不弹窗提示，直接清空会话跳转登录页。
- error_message 由装置端填写中文文本，作为兜底提示。管理端在已知结果码场景下使用本表定义的提示文案，不依赖 error_message。
- 同一常量名可出现在多个模块中（如 FORMAT_ERROR 分别为 0x0402 策略导入和 0x0602 系统管理），数字值不同，含义按模块上下文区分。
- 新增结果码必须在对应模块段内分配编号，不得跨段使用。

### 查询类命令错误处理约定

查询类命令（CMD_LIST_WHITELIST、CMD_GET_CONNECTED_DEVICES、CMD_GET_FILE_POLICY、CMD_QUERY_LOGS、CMD_GET_SYSTEM_INFO、CMD_LIST_USERS、CMD_AUTH_STATUS_QUERY、CMD_GET_MACHINE_CODE）的错误处理：

- 全局错误（会话无效、未授权、无权限等）：装置端返回 RspCommon（0xFF00）携带 result_code，管理端通过消息类型码区分专用响应和通用错误响应。
- 业务正常但无数据：返回专用响应，列表字段为空数组。
- 装置内部错误（如数据库查询失败）：装置端返回 RspCommon 携带 INTERNAL_ERROR（0x0007）。
