# 05 数据流设计

---

## 1. 核心数据流：U 盘准入与映射

```
USB 插入事件（udev）
    ↓
S01 USB 设备识别与映射引擎：读取 USB 描述符、接口类、接口子类、协议号、端点能力，识别普通大容量存储设备、键盘、鼠标或不支持设备
    ├── 大容量存储设备（U盘）
    │       ↓
    │   S05 白名单查询（序列号）
    │       ├── 不在白名单 ────────────────────────→ 禁止映射，记录日志（S07）
    │       └── 在白名单
    │               ↓
    │           kernel mount 到 /mnt/usb_raw
    │               ↓
    │           S03 病毒扫描（ClamAV，全盘）
    │               ├── 扫描中 → 禁止映射，等待扫描完成
    │               ├── 扫描失败/病毒库不可用 ───→ 禁止映射，记录恶意代码日志（S07）
    │               ├── 发现病毒 → S03 输出病毒文件路径列表，记录恶意代码日志（S07）
    │               │         ─→ S04 构建受控文件树时，对命中文件在虚拟 FAT32 目录项中添加 [病毒禁止访问] 前缀（不修改原始 U 盘文件名）
    │               │         ─→ 可映射（非命中文件可访问，命中文件由 NBD 层返回 I/O error）
    │               └── 未发现病毒 → 继续
    │                       ↓
    │                   S04 构建受控文件树（基于 /mnt/usb_raw，文件访问控制策略生效）
    │                       ↓
    │                   NBD 虚拟块设备服务器启动（FAT32 扇区按需生成，/dev/nbdX）
    │                       ↓
    │                   f_mass_storage LUN 绑定 /dev/nbdX
    │                       ↓
    │                   OTG 映射到受控主机（受控主机可见受控 U 盘）
    │                       ↓
    │                   记录 USB 审计日志（S07）
    │
    ├── 键盘设备
    │       ↓
    │   HID 接管，阻断透传到受控主机
    │       ↓
    │   等待用户在键盘上顺序输入 1234（键值序列监听，不透传）
    │       ├── 输入正确 → OTG 映射键盘到受控主机，记录日志（S07）
    │       ├── 输入错误 → 不映射，清空当前输入序列并继续等待，记录验证失败日志（S07）
    │       └── 键盘拔出 → 结束本次验证，释放 HID 接管和输入缓冲，记录拔出日志（S07）
    │
    ├── 鼠标设备
    │       ↓
    │   自动 OTG 映射到受控主机，记录日志（S07）
    │
    └── 未知设备 / 不支持设备 / 描述符与能力不一致
            ↓
        默认禁止映射并记录 USB 审计日志（S07）
```

**设备安全准入原则：**

- 不按设备名称、厂商字符串、产品字符串或单一序列号判断整台设备可信。
- 白名单只对已识别为普通大容量存储设备的对象生效，不自动放行键盘、鼠标、未知或不支持设备。
- 设备描述符、接口类、接口子类、协议号、端点能力与预期类型不一致时，按伪装或未知设备处理，默认拒绝映射并记录 USB 审计日志。

## 2. 文件访问控制数据流

```
受控主机文件访问（Windows Explorer 打开 / 读取 / 写入文件）
    ↓
USB Mass Storage SCSI 命令（READ(10) / WRITE(10)，按 LBA 扇区地址）
    ↓
f_mass_storage gadget → /dev/nbdX
    ↓
内核 NBD 客户端 → NBD 服务器（S04，Rust 进程）
    ↓
扇区地址 → 虚拟 FAT32 布局解析 → 对应 /mnt/usb_raw 文件路径 + 块内偏移
    ↓
策略优先级检查（5 级，短路求值）：

    1. 病毒标记检查（文件名含 [病毒禁止访问] 前缀）
           命中 → 文件在目录中可见，读取对应数据扇区时返回 I/O error

    2. 可执行程序访问控制开关开启 且 文件为 4 种可执行类型之一（dll / exe / PE / ELF，基于文件实际格式识别）
           命中 → 文件在目录中可见，读取对应数据扇区时返回 I/O error

    3. 文件后缀命中黑名单（大小写不敏感）
           命中 → 文件在目录中可见，读取对应数据扇区时返回 I/O error

    4. 介质自动读取控制开关开启 且 操作为自动读取（Autorun 等）
           命中 → 对自动读取请求返回 I/O error（阻断 Autorun 触发）

    5. 设备权限为只读 且 操作为 WRITE
           命中 → f_mass_storage write-protect flag 已置位，整盘写保护（SCSI 层拒绝，非文件级）

    全部未命中 → 从 /mnt/usb_raw 按需读取原始文件内容，返回真实扇区数据

    命中 1-4 的事件记录到 USB 审计日志（S07）
```

**策略优先级与受控主机实际效果：**

| 优先级 | 策略 | 触发条件 | 受控主机可见效果 |
|---|---|---|---|
| 1（最高） | 病毒标记阻断 | 文件名含 `[病毒禁止访问]` 前缀 | 目录可见，打开时报读取文件出错 / I/O error |
| 2 | 可执行程序访问控制 | 开关开启 + 文件为可执行类型 | 目录可见，打开时报读取文件出错 / I/O error |
| 3 | 文件类型黑名单 | 文件后缀命中黑名单 | 目录可见，打开时报读取文件出错 / I/O error |
| 4 | 介质自动读取控制 | 开关开启 + 操作为自动读取 | 自动读取请求返回 I/O error，Autorun 触发被阻断 |
| 5（最低） | 白名单权限只读 | 设备权限为只读 + 写操作 | 整盘写保护（SCSI Write Protect）|

> **说明**：FAT32 无文件级 ACL，受控主机直接消费 FAT32 扇区。优先级 1-4 的阻断效果由 NBD server 在命中策略的数据扇区 READ 请求上返回 I/O error 实现；受控主机可见文件目录项，但打开文件时报“读取文件出错 / 输入输出错误”等系统提示。

## 3. 管理端登录、授权与配置读取数据流

```
管理端用户填写账号/密码/装置IP，点击登录
    ↓
装置IP连接：建立 TCP 连接 → TLS 握手（校验证书指纹）
    ├── 连接超时/拒绝 → 提示"连接装置失败，请检查装置IP"
    ├── 证书指纹不匹配 → 提示"版本不兼容，请升级管理端"
    └── 连接成功
            ↓
        C03 发送登录请求（用户名/密码）
            ↓
        装置端 S08 校验账号密码和锁定状态
            ├── 用户名不存在/密码错误 → 提示"用户名或密码错误"，装置侧失败计数 +1
            ├── 账号已锁定 → 提示"用户已被锁定，请过5分钟后重试"
            └── 鉴权通过 → 装置侧失败计数清零，返回用户名/角色/临时会话 token
    ↓
装置端检查授权状态
    ↓
    ├── 未授权 + 任一有效账号 → 管理端跳转独立授权页
    │       └── 授权成功 → 自动返回登录页，用户重新输入 IP/用户名/密码登录
    └── 已授权 → C04 按当前角色拉取该角色可见页面所需装置数据：
                    - 操作员：白名单列表、文件访问控制策略（三开关 + 黑名单）
                    - 审计员：无需预拉取（进入日志页后按查询条件拉取）
                    - 系统管理员：系统版本、病毒库版本、授权信息（到期时间）、设备描述、用户列表
                    ↓
                缓存到 C04 内存（不落盘）
                    ↓
                C02 设置全局状态：已连接 + 当前角色
                    ↓
                跳转对应角色默认页面
```

## 4. 主动登出数据流

```
已登录用户点击登出
    ↓
C03 发送 CMD_LOGOUT（0x0009，session_token）到装置端
    ↓
装置端 S10 校验会话 token 并路由到 S08
    ↓
S08 失效当前会话 token
    ↓
S07 写入操作日志（用户登出）
    ↓
装置端返回通用响应
    ↓
C02/C04 清空当前进程内会话状态和当前页面进程内状态
    ↓
C01 返回登录页
```

**异常处理：** 如果用户点击登出时管理端与装置已断开连接，管理端无法通知装置侧失效 token；此时管理端仍需清空本地进程内会话状态并返回登录页，装置侧会话按自身超时规则失效。

## 5. 策略导入数据流

```
管理端：操作员上传 .bin 策略文件
    ↓
Electron 主进程读取文件字节流
    ↓
C03 发送导入策略消息（含文件字节流）到装置端
    ↓
装置端 S06 导入策略：
    1. 校验文件头魔数和版本
    2. 校验算法标识、SM2 签名和 SM3 摘要
    3. 使用 SM4 解密策略密文
    4. 反序列化策略内容
    5. 开启 SQLite 事务
    6. 整体覆盖白名单表和文件访问控制策略表
    7. 提交事务
        ├── 任一步骤失败 → 事务回滚，原策略完整保留，返回失败原因
        └── 全部成功 → 提交，通知 S04 重新加载策略，返回成功
            ↓
        管理端收到成功响应 → C04 更新当前页面进程内状态 → 提示导入成功
```

## 7. 管理端与装置断开连接恢复数据流

```
C03 检测到 TCP 连接断开（心跳超时 / 网络异常）
    ↓
C03 触发连接断开事件，通知 C02
    ↓
C02 更新 Pinia 全局状态：连接状态 = 已断开
    ↓
当前页面保持展示当前页面进程内状态，不清空、不跳登录页
    ↓
用户点击任何查询、导出、导入、保存、删除、升级、授权等下发动作
    ↓
C03 立即返回连接不可用错误，页面提示"装置已断开，请重新连接后再操作"

用户重新连接装置
    ↓
C03 重新建立 TCP 连接 → TLS 握手
    ├── 连接失败 → 保持断开状态，用户可再次尝试
    └── 连接成功
            ↓
        C03 发送会话校验请求（当前内存中的 session_token）
            ├── 会话有效 → 停留当前页面，C04 全量重新拉取该角色可见页面装置数据，UI 刷新
            └── 会话失效 → C02/C04 清空进程内会话状态，返回登录页重新登录
```

## 8. 系统程序升级数据流

```
管理端：系统管理员上传 .bin 升级包
    ↓
Electron 读取文件
    ↓
C03 发送系统升级消息（含升级包字节流）
    ↓
装置端 S09：
    1. 校验文件格式和完整性（SHA256 校验和）
    2. 比对版本号（升级包版本必须大于当前系统版本，不允许降版本）
    3. 将升级包写入临时目录
    4. 将新版本写入独立文件并完成校验
    5. 使用原子切换方式替换当前运行版本
        ├── 失败 → 保留原版本，返回失败原因，记录操作日志（S07）
        └── 成功 → 返回成功响应，记录操作日志（S07）
                ↓
            装置端重启服务（systemd restart）
                ↓
            管理端收到连接断开，显示连接断开提示
            提示"升级完成，请重新连接"
```

## 9. 病毒库升级数据流

```
管理端：系统管理员上传 .zip 病毒库升级包
    ↓
Electron 读取文件
    ↓
C03 发送病毒库升级消息（含升级包字节流）
    ↓
装置端 S09：
    1. 校验文件格式（.zip）
    2. 比对版本号（升级包版本必须高于当前病毒库版本，不允许降版本）
       版本号从 V3.0.0.0 开始递增，递增时逢数字 4 跳过（如 V3.0.0.3 → V3.0.0.5）
    3. 解压并校验病毒库文件完整性
    4. 替换 ClamAV 病毒库文件
    5. 通知 clamd 重新加载病毒库
        ├── 任一步骤失败 → 保留旧版本病毒库，尝试回滚到升级前可用状态，返回失败原因，记录操作日志（S07）
        └── 成功 → 更新病毒库版本和更新时间，返回成功响应，记录操作日志（S07）
                ↓
            管理端收到成功响应 → 刷新页面病毒库版本和更新时间
```

**与系统程序升级的区别：** 病毒库升级不重启装置服务，不断开管理端连接；升级后新插入的 U 盘使用新病毒库扫描，已映射 U 盘不受影响。

```
                ↓

                ↓
            提示导出成功
```

```

    ↓

                - 操作日志：OperationLogYYYYMMDDHHMMSS.zip
```

## 12. 日志清理数据流

```
审计员在日志管理页点击清理，选择时间范围
    ↓
前端校验：日期选择器禁用半年内的日期（从源头防止误选）
    ↓
二次确认弹窗
    ↓
C03 发送日志清理请求（含日志类型/清理截止时间/会话 token）
    ↓
装置端 S07：
    1. 校验清理截止时间（半年内日志禁止清理）
    2. 删除指定类型中截止时间之前的日志记录
        ├── 失败 → 返回失败原因
        └── 成功 → 记录操作日志（S07），返回成功响应
                ↓
            管理端刷新日志列表
```

**装置侧自动滚动覆盖规则：** 装置整体存储使用率超过 80% 时，写入某类日志只覆盖该类别最旧记录，不跨类别覆盖。此规则由装置端 S07 在日志写入时自动执行，不依赖管理端触发。

```
                ↓
            管理端关闭弹窗，提示修改成功
```

## 14. 操作日志写入数据流

以白名单添加为例，说明操作日志的写入时机：

```
操作员点击"添加设备"，填写表单，确认
    ↓
C03 发送白名单添加消息到装置端
    ↓
装置端 S05 执行添加操作
    ├── 失败 → 返回失败原因
    └── 成功 → 返回成功

装置端写入操作日志（S07）：
    └── 成功 → 操作账号 / 角色 / 操作类型="添加白名单设备" /
                操作对象=序列号 / 权限设置 / 结果=成功 / 时间
    └── 失败 → 操作账号 / 角色 / 操作类型="添加白名单设备" /
                操作对象=序列号 / 结果=失败 / 失败原因=...

管理端收到响应后只更新内存页面状态或展示失败提示，不落盘保存日志。
```

**原则：操作日志由装置端在处理管理指令时写入，无论成功失败均记录；管理端不持久化操作日志。**
