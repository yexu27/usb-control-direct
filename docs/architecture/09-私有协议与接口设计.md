# 09 私有协议与接口设计

---

## 说明

本系统管理端与装置端之间使用自定义私有协议通信（TCP + TLS + 二进制消息帧）。本文档定义：

1. 协议帧格式
2. 连接、鉴权与登出流程
3. 所有消息类型（命令码）及其载荷结构

载荷使用 Protocol Buffers 序列化，`.proto` 文件在 `proto/` 目录中定义，两端共享。

---

## 1. 协议帧格式

每条消息由一个固定头部 + 可变长度载荷组成：

```
┌─────────────────────────────────────────────────────────┐
│  [4字节] Magic Number  │  0x55534243 ("USBC")            │
├─────────────────────────────────────────────────────────┤
│  [4字节] Message Type  │  uint32，消息类型码               │
├─────────────────────────────────────────────────────────┤
│  [4字节] Sequence ID   │  uint32，请求序列号（响应与请求匹配）│
├─────────────────────────────────────────────────────────┤
│  [4字节] Payload Length│  uint32，载荷字节数（最大 128MB）  │
├─────────────────────────────────────────────────────────┤
│  [4字节] CRC32         │  uint32，Payload 的 CRC32 校验和  │
├─────────────────────────────────────────────────────────┤
│  [N字节] Payload       │  Protobuf 序列化的载荷，可以为空   │
└─────────────────────────────────────────────────────────┘
```

**约定：**

- Magic Number 校验失败，立即关闭连接
- Payload Length 超过 128MB，立即关闭连接
- CRC32 校验失败，丢弃该消息并记录错误日志；连续 3 次校验失败，关闭连接
- CRC32 算法使用 IEEE 802.3 标准多项式（0xEDB88320），Payload 为空时 CRC32 值为 0x00000000
- 所有整数大端字节序（Big Endian）

## 2. 连接、登录、登出与授权流程

**登录页三项输入：装置 IP、用户名、密码。管理端不保存默认 IP，不记住最近 IP。**

1. 装置 IP → 发起 TCP 连接 + TLS 握手，校验证书指纹
2. 用户名 + 密码 → 发送到装置端，由装置侧用户表校验
3. 鉴权通过后，装置端检查授权状态并返回角色、会话 token 和授权分支

```
管理端                                             装置端
  │                                                  │
  │─── TCP Connect ─────────────────────────────────→│
  │←── TLS Handshake ───────────────────────────────→│
  │    (装置侧使用构建时内置的自签名证书；                 │
  │     管理端安装包内置对应证书的 SHA256 指纹；           │
  │     TLS 握手后校验指纹是否与安装包内置值一致，          │
  │     不一致则拒绝连接并提示                           │
  │     "版本不兼容，请升级管理端")                      │
  │                                                  │
  │─── CMD_LOGIN(username, password) ───────────────→│
  │←── RSP_LOGIN(token, role, auth...) ──────────────│
  │                                                  │
  [未授权 + 任一有效账号分支]                            │
  │─── CMD_GET_MACHINE_CODE ────────────────────────→│
  │←── RSP_MACHINE_CODE ──────────────────────────── │
  │─── CMD_UPLOAD_LICENSE ──────────────────────────→│
  │←── RSP_UPLOAD_LICENSE ────────────────────────── │
  │                                                  │
  [已授权分支：发送带 token 的管理命令]                   │
  │─── CMD_BUSINESS(session_token, ...) ────────────→│
  │←── RSP_BUSINESS_RESPONSE ─────────────────────── │
  │                                                  │
  [主动登出]                                          │
  │─── CMD_LOGOUT(session_token) ───────────────────→│
  │←── RSP_COMMON ────────────────────────────────── │
```

**说明：装置端负责用户鉴权、会话 token、授权状态和角色权限校验。管理端只在进程内保存当前 token 和角色，不落盘保存账号、token 或最近 IP。装置未授权时，系统管理员、操作员、审计员任一有效账号鉴权通过后均进入独立授权页；授权成功后返回登录页并要求重新登录。进入主系统后的系统管理授权信息管理仍仅系统管理员可调用。用户主动登出时，管理端应调用 `CMD_LOGOUT` 使装置侧当前会话 token 失效，并清空管理端进程内会话状态。**

## 3. 消息类型码定义

消息类型码分区间分配：


| 区间              | 用途                 |
| --------------- | ------------------ |
| 0x0001 - 0x00FF | 连接、登录、登出与授权管理      |
| 0x0100 - 0x01FF | 白名单管理              |
| 0x0200 - 0x02FF | 文件访问控制策略           |
| 0x0300 - 0x03FF | 策略导入导出             |
| 0x0400 - 0x04FF | 日志查询               |
| 0x0500 - 0x05FF | 系统管理（升级/授权/设备描述）   |
| 0x0600 - 0x06FF | 用户管理               |
| 0x0700 - 0x07FF | 当前未分配，本版本不使用主动通知消息 |
| 0xFF00          | 通用响应 RSP_COMMON    |
| 0xFF01          | 心跳请求 CMD_HEARTBEAT |
| 0xFF02          | 心跳响应 RSP_HEARTBEAT |

## 4. 连接、登录、登出与授权消息

### CMD_LOGIN（0x0001）

登录装置。无需 session token。

**请求载荷：**

```protobuf
message CmdLogin {
    string username = 1;
    string password = 2;
}
```

**响应 RSP_LOGIN（0x0002）：**

```protobuf
message RspLogin {
    bool success = 1;
    string session_token = 2;      // 成功时返回，仅管理端进程内保存
    string username = 3;
    string role = 4;               // admin / operator / auditor
    bool authorized = 5;           // 兼容性字段，true 表示 auth_status=authorized
    int64 auth_expire_time = 6;     // Unix 秒；0 表示未授权 / 无到期时间
    string device_description = 7;
    int32 result_code = 8;          // 0x0101 / 0x0004 / 0x0102（见统一结果码表）
    string error_message = 9;
    string auth_status = 10;       // authorized / unauthorized / expired / failed
}
```

**角色标识映射：**

协议层使用字符串标识，数据库层（T08）使用整数存储，映射关系如下：

| 协议值（string） | 数据库值（T08.role INTEGER） | 含义 |
|---|---|---|
| admin | 0 | 系统管理员 |
| operator | 1 | 操作员 |
| auditor | 2 | 审计员 |

装置端在协议响应中将 T08.role 整数转换为上述字符串；管理端只使用字符串标识，不感知整数编码。

登录成功时装置侧清零失败计数；登录失败时装置侧递增失败计数，连续 5 次锁定 5 分钟；锁定超时自动解除后清零失败计数。

### CMD_LOGOUT（0x0009）

登出并使会话 token 失效。

**请求载荷：**

```protobuf
message CmdLogout {
    string session_token = 1;
}
```

**响应：** 通用响应 RSP_COMMON

**处理规则：**

- `session_token` 有效时，装置侧立即使该会话 token 失效。
- `session_token` 无效或已过期时，返回 `UNAUTHENTICATED`（0x0001）；管理端仍需清空本地进程内会话状态并返回登录页。
- 登出动作由装置侧记录操作日志，操作日志类型为“登录认证”。

### CMD_GET_CONNECTED_DEVICES（0x0102）

获取当前连接装置中可添加白名单的 USB 设备列表。仅返回当前仍连接、序列号去除首尾空白后非空、已识别为普通大容量存储且 `admission_status=addable` 的候选设备；键盘、鼠标、unknown、unsupported、疑似伪装、空序列号和已在白名单中的设备不出现在响应中。

**请求载荷：**

```protobuf
message CmdGetConnectedDevices {
    string session_token = 1;
}
```

**响应 RSP_CONNECTED_DEVICES（0x0103）：**

```protobuf
message ConnectedDevice {
    string serial_number = 1;
    string device_name = 2;
    string vid = 3;
    string pid = 4;
    int64 capacity_bytes = 5;
    string device_type = 6;       // storage / keyboard / mouse / unsupported / unknown
    string interface_type = 7;    // mass_storage / hid_keyboard / hid_mouse / unsupported / unknown
    string admission_status = 8;  // addable / unsupported / spoof_suspected / blocked
    string fail_reason = 9;       // 不可添加或禁止映射原因
}

message RspConnectedDevices {
    repeated ConnectedDevice devices = 1;
}
```

响应中的每一项均固定为 `device_type=storage`、`interface_type=mass_storage`、`admission_status=addable`、`fail_reason=""`。

### CMD_ADD_WHITELIST（0x0104）

添加白名单设备。

**请求载荷：**

```protobuf
message CmdAddWhitelist {
    string session_token = 1;
    string serial_number = 2;
    string vid = 3;
    string pid = 4;
    string device_name = 5;
    int64 capacity_bytes = 6;
    string permission = 7;
    string description = 8;
    string add_method = 9;
    string device_type = 10;     // storage
}
```

**响应：** 通用响应 RSP_COMMON

**安全准入约束：**

- 白名单只对已识别为普通大容量存储设备的对象生效。
- 装置端根据 USB 描述符、接口类、接口子类、协议号和端点能力判断设备类型；不得仅依据设备名称、厂商字符串、产品字符串或序列号放行。
- `device_type` 或 `interface_type` 为 `unknown`、`unsupported`，以及描述符与接口能力不一致或疑似伪装的设备，不允许通过白名单接口添加为可映射 U 盘。
- `add_method` 取值为 `device` 或 `management`。`device` 表示装置端添加；服务端收到添加请求后必须按序列号重新确认设备仍连接，并再次校验 `device_type=storage`、`interface_type=mass_storage`、`admission_status=addable`。设备在位读锁须持续持有到白名单数据库提交完成，避免拔出事件介入复核与提交之间。若设备已移除，返回 `VALIDATION_FAILED` 和“设备已移除，请重新插入后再添加”。装置端添加忽略客户端提交的 VID、PID、设备名称、容量和设备类型，统一使用当前设备会话的真实识别字段。`management` 表示管理端添加，设备信息来自管理主机本机 USB 大容量存储设备枚举结果，VID/PID 允许为空，但 `device_type` 必须为 `storage`。
- 添加请求的序列号去除首尾空白后为空时返回 `SERIAL_NUMBER_EMPTY (0x0201)`；非存储设备返回 `DEVICE_NOT_STORAGE (0x0204)`；疑似伪装设备返回 `DEVICE_SPOOF_SUSPECTED (0x0205)`；unknown / unsupported 返回 `DEVICE_UNSUPPORTED (0x0206)`，提示文案遵循《统一结果码表》定义。
- 管理端添加仅写入白名单记录，不跳过设备后续插入装置端时的 S01 真实识别；非普通大容量存储、unknown、unsupported 或疑似伪装设备不得因序列号命中白名单而被映射。

## 8. 日志查询消息

查询装置侧日志（USB 审计日志、恶意代码检测日志或操作日志）。

    int64 id = 1;

    string role = 4;
    string log_category = 5;       // 操作日志分类，对应 T10.log_type，用于页面“操作日志类型”列展示

## 12. 心跳消息

### CMD_HEARTBEAT（0xFF01）/ RSP_HEARTBEAT（0xFF02）

心跳保活，管理端每 30 秒发送一次，装置端收到后立即回复。超过 3 次无响应视为连接断开。

**请求载荷：** 空

**响应载荷：** 空

### 会话超时规则

装置端维护每个 session token 的最后业务请求时间。超过 5 分钟无业务请求时，装置端标记该会话过期。管理端下次发送业务请求时，装置端返回 `UNAUTHENTICATED`（0x0001），管理端清空进程内会话状态并跳转登录页。

- 心跳（CMD_HEARTBEAT）为连接层保活消息，不更新会话最后业务请求时间，不重置超时计时器。
- 所有携带 session_token 的业务请求更新会话最后业务请求时间。
- 会话过期后，管理端调用 CMD_LOGOUT 返回 `UNAUTHENTICATED`（0x0001），管理端按正常登出处理。

### 请求超时规则

管理端发送请求后，按消息类型分档管理等待超时。超时时长考虑局域网网络波动（交换机转发、网络拥塞、TCP 丢包重传）和装置端 ARM 处理能力。

**超时分档：**


| 档位   | 超时  | 适用范围                                                                                                             | 说明                             |
| ---- | --- | ---------------------------------------------------------------------------------------------------------------- | ------------------------------ |
| 连接超时 | 15秒 | TCP 连接 + TLS 握手                                                                                                 | 局域网拥塞时 SYN 重传和 TLS 多轮握手        |
| 普通指令 | 15秒 | 0x0001-0x02FF, 0x0404, 0x0500, 0x0504, 0x0600-0x06FF                                                            | 查询、增删改、开关切换、登录登出、用户管理、日志清理     |
| 文件传输 | 300秒 | 0x0007（授权上传）, 0x0300（策略导出）, 0x0302（策略导入）, 0x0402（日志导出）, 0x0502（系统升级）, 0x0503（病毒库升级） | 大文件传输 + 装置端校验/解密/落盘/重载，最长等待 5 分钟 |


**超时处理策略：**

- 请求超时后**不断开 TCP 连接**。超时可能是装置端处理慢，不代表连接失效；连接是否存活由心跳机制独立判断。
- 管理端基于协议帧 Sequence ID 管理超时。超时后该 Sequence ID 标记为已废弃，迟到的响应到达时直接丢弃，不触发业务回调。
- **写入类指令和文件传输类指令不自动重试**，避免重复执行产生副作用。超时后提示用户，由用户决定是否重试。

**查询类指令自动重试：**

网络波动可能导致单次请求偶发超时。以下只读查询指令超时后自动重试 1 次（使用新 Sequence ID），两次均超时才提示用户：

- CMD_LIST_WHITELIST（0x0100）
- CMD_GET_CONNECTED_DEVICES（0x0102）
- CMD_GET_FILE_POLICY（0x0200）
- CMD_QUERY_LOGS（0x0400）
- CMD_GET_SYSTEM_INFO（0x0500）
- CMD_LIST_USERS（0x0600）
- CMD_AUTH_STATUS_QUERY（0x0003）

**文件传输类指令超时提示：**

文件传输类指令超时时，管理端需区分两种场景给出不同提示：

| 场景             | 判断依据              | 提示文案                                   |
| -------------- | ----------------- | -------------------------------------- |
| 超时但连接仍在        | TCP 连接存活，心跳正常     | "操作超时，装置可能仍在处理，请稍后确认结果再决定是否重试"         |
| 超时且连接已断开       | 心跳超时或 TCP 断开已触发   | "网络连接中断，请重新连接后查看装置状态"                  |

## 13. 通用响应 RSP_COMMON（0xFF00）

所有无特定响应载荷的命令使用通用响应（消息类型码 0xFF00）：

```protobuf
message RspCommon {
    bool success = 1;
    int32 result_code = 2;       // 失败时的结果码（见统一结果码表）
    string error_message = 3;   // 失败时的提示文本（中文，直接展示给用户）
}
```

**常用结果码（完整列表见统一结果码表）：**


| 结果码 | 常量名                      | 含义                   |
| ------ | ------------------------ | -------------------- |
| 0x0006 | DEVICE_BUSY              | 装置正在执行其他操作（如扫描中）     |
| 0x0005 | VALIDATION_FAILED        | 请求参数校验失败             |
| 0x0202 | ALREADY_EXISTS           | 资源已存在（如重复序列号）        |
| 0x0203 | NOT_FOUND                | 目标资源不存在              |
| 0x0601 | VERSION_TOO_LOW          | 升级包版本低于当前版本          |
| 0x0406 | VERSION_INCOMPATIBLE     | 策略文件或升级包版本不兼容        |
| 0x0605 | VERSION_NUMBER_FORBIDDEN | 病毒库版本号命中逢 4 跳过规则     |
| 0x0402 | FORMAT_ERROR             | 文件格式错误               |
| 0x0002 | UNAUTHORIZED             | 装置未授权                |
| 0x0001 | UNAUTHENTICATED          | session token 无效或已过期 |
| 0x0003 | PERMISSION_DENIED        | 当前角色无权限              |
| 0x0004 | ACCOUNT_LOCKED           | 账号已锁定                |
| 0x0007 | INTERNAL_ERROR           | 装置内部错误               |
