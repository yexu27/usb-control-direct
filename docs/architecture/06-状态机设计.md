# 06 状态机设计

---

## 1. U 盘设备状态机

每个接入的 U 盘设备维护一个独立状态机实例，由 S01 管理。

```
                         ┌────────────────┐
                         │  USB 插入事件   │
                         └───────┬────────┘
                                 ↓
                         ┌───────────────┐
                         │   DETECTED    │ 已检测
                         └───────┬───────┘
                                 ↓ 读取描述符/接口/端点能力
                         ┌───────────────┐
                         │  CLASSIFYING  │ 识别中
                         └───────┬───────┘
                                 │
                 ┌───────────────┴───────────────┐
                 │ 非存储/未知/伪装                │ 大容量存储接口
                 ↓                               ↓
         ┌───────────────┐                  白名单查询
         │   REJECTED    │                 ┌─────┴─────┐
         │   禁止映射 ×   │                 │            │
         └───────┬───────┘              不在白名单   在白名单
                 │                         │           │
                 │                         ↓           ↓
                 │                 ┌────────────┐ ┌────────────┐
                 │                 │  BLOCKED   │ │  SCANNING  │
                 │                 │  禁止映射 × │ │  扫描中     │
                 │                 └─────┬──────┘ └─────┬──────┘
                 │                       │              │
                 │                       │  ┌───────────┴────────┐
                 │                       │  │ 扫描失败            │ 扫描完成
                 │                       │  ↓                    │
                 │                 ┌────────────┐          ┌─────┴─────┐
                 │                 │ SCAN_FAILED│          │           │
                 │                 │ 禁止映射 ×  │         未发现      发现病毒
                 │                 └─────┬──────┘          ↓           ↓
                 │                       │           ┌─────────┐ ┌──────────┐
                 │                       │           │  CLEAN  │ │ INFECTED │
                 │                       │           │  干净    │ │ 检出病毒  │
                 │                       │           └────┬────┘ └────┬─────┘
                 │                       │                │           │
                 │                       │                ↓           ↓
                 │                       │              S04 构建受控文件树
                 │                       │              文件访问控制策略生效
                 │                       │              INFECTED: 病毒文件
                 │                       │              在目录项添加前缀标记
                 │                       │                │           │
                 │                       │                ↓           ↓
                 │                       │            NBD 虚拟 exFAT 块设备启动
                 │                       │                │           │
                 │                       │                ↓           ↓
                 │                       │            OTG gadget 映射到受控主机
                 │                       │                │           │
                 │                       │           ┌────┴───────────┤
                 │                       │           │ 成功            │ 失败
                 │                       │           ↓                ↓
                 │                       │    ┌─────────────┐  ┌────────────┐
                 │                       │    │   MAPPED    │  │ MAP_FAILED │
                 │                       │    │ 受控主机可见  │  │ 映射失败 ×  │
                 │                       │    │ 受控 U 盘    │  └─────┬──────┘
                 │                       │    │ NBD 按 5 级  │        │
                 │                       │    │ 策略响应读写  │        │
                 │                       │    └──────┬──────┘        │
                 │                       │           │               │
                 ↓                       ↓           ↓               ↓
         ┌────────────────────────────────────────────────────────────────┐
         │             任意状态 + USB 拔出 → REMOVED                        │
         │       MAPPED: 卸载 /mnt/usb_raw，停止 NBD，清理 OTG               │
         │       其他状态: 清理已分配资源                                      │
         └────────────────────────────────────────────────────────────────┘
```

**状态转换表：**


| 当前状态        | 事件                        | 下一状态        | 动作                                                        |
| ----------- | ------------------------- | ----------- | --------------------------------------------------------- |
| DETECTED    | USB 插入                    | CLASSIFYING | 读取设备描述符、配置描述符、接口类、接口子类、协议号和端点能力                           |
| CLASSIFYING | 非存储接口、未知接口、描述符与能力不一致或疑似伪装 | REJECTED    | 默认拒绝映射，记录 USB 审计日志                                        |
| CLASSIFYING | 白名单查询：不在白名单               | BLOCKED     | 记录 USB 审计日志（未认证设备）                                        |
| CLASSIFYING | 白名单查询：在白名单                | SCANNING    | Linux 挂载层挂载到 `/mnt/usb_raw`，提交病毒扫描任务（S03）               |
| CLASSIFYING | 在白名单但 Linux 挂载层挂载失败     | SCAN_FAILED | 记录 USB 审计日志（挂载失败，禁止映射）                                    |
| SCANNING    | 扫描完成：未发现病毒                | CLEAN       | 记录恶意代码检测日志（未发现）                                           |
| SCANNING    | 扫描完成：发现病毒                 | INFECTED    | S03 输出病毒文件路径列表，记录恶意代码检测日志                                 |
| SCANNING    | 扫描失败/病毒库不可用               | SCAN_FAILED | 记录恶意代码检测日志（扫描失败）                                          |
| CLEAN       | NBD 受控块设备启动 + OTG 映射成功    | MAPPED      | 记录 USB 审计日志（映射成功）                                         |
| CLEAN       | NBD 启动失败或 OTG 映射失败        | MAP_FAILED  | 记录 USB 审计日志（映射失败），清理已分配资源                                 |
| INFECTED    | NBD 受控块设备启动 + OTG 映射成功    | MAPPED      | 记录 USB 审计日志（含病毒，映射成功）                                     |
| INFECTED    | NBD 启动失败或 OTG 映射失败        | MAP_FAILED  | 记录 USB 审计日志（映射失败），清理已分配资源                                 |
| REJECTED    | USB 拔出                    | REMOVED     | 记录 USB 审计日志（拔出）                                           |
| BLOCKED     | USB 拔出                    | REMOVED     | 记录 USB 审计日志（拔出）                                           |
| SCAN_FAILED | USB 拔出                    | REMOVED     | 记录 USB 审计日志（拔出）                                           |
| MAP_FAILED  | USB 拔出                    | REMOVED     | 记录 USB 审计日志（拔出），清理残留资源                                    |
| MAPPED      | USB 拔出                    | REMOVED     | 卸载 `/mnt/usb_raw`，停止 NBD server，清理 OTG 映射，记录 USB 审计日志（拔出） |
| 任意状态        | 异常（内核错误等）                 | REMOVED     | 清理资源，记录错误日志                                               |


**注意：**

- REJECTED、BLOCKED、SCAN_FAILED 和 MAP_FAILED 状态下设备不映射到受控主机
- CLEAN 和 INFECTED 为瞬态状态，系统进入后立即触发 S04 构建受控文件树和 NBD/OTG 映射流程，无驻留等待
- 白名单只放行已识别为普通大容量存储设备的对象，不因设备名称、厂商字符串、产品字符串或序列号相似而放行键盘、鼠标、未知或不支持设备
- INFECTED 状态下允许映射，但命中病毒的文件在虚拟 exFAT 视图中显示为 `[病毒禁止访问]` 前缀且 size=0；对该文件的读、写、修改、删除、重命名、执行、复制等操作均禁止，不向受控主机暴露真实内容，不修改真实 U 盘原始文件名和内容

## 2. 键盘设备状态机

```
                ┌────────────────┐
                │  USB 插入事件   │
                └───────┬────────┘
                        ↓
                ┌───────────────┐
                │  KB_DETECTED  │ 已检测为键盘
                └───────┬───────┘
                        │ 接管 HID
                        ├─────────────────┐
                    接管成功          接管失败
                        ↓                 ↓
                ┌───────────────┐  ┌──────────────┐
                │  KB_WAITING   │  │ KB_REJECTED  │ 等待拔出
                └───────┬───────┘  └──────────────┘
                        │ 等待用户输入（无固定超时）
          ┌─────────────┼─────────────┐
          │ 顺序输入 1234 │ 输入错误     │ USB 拔出
          ↓              ↓            ↓
  ┌───────────────┐  KB_WAITING   ┌─────────────┐
  │   KB_MAPPED   │  清空输入序列   │  KB_REMOVED │
  │   键盘已映射    │  继续等待      │  验证结束    │
  └───────┬───────┘               └─────────────┘
          │
          ↓ USB 拔出
  ┌─────────────┐
  │  KB_REMOVED │
  └─────────────┘
```

**状态转换表：**


| 当前状态        | 事件             | 下一状态        | 动作                                |
| ----------- | -------------- | ----------- | --------------------------------- |
| KB_DETECTED | HID 接管成功       | KB_WAITING  | 阻断键盘输入透传到受控主机，开始监听键值              |
| KB_DETECTED | HID 接管失败       | KB_REJECTED | 记录 USB 审计日志（键盘 HID 接管失败），不映射到受控主机 |
| KB_DETECTED | USB 拔出         | KB_REMOVED  | 记录 USB 审计日志（拔出）                   |
| KB_WAITING  | 顺序输入 1234 完整匹配 | KB_MAPPED   | OTG 映射键盘到受控主机，记录 USB 审计日志（验证通过）   |
| KB_WAITING  | 输入错误（非预期键值）    | KB_WAITING  | 清空当前输入序列，继续等待，记录验证失败日志            |
| KB_WAITING  | USB 拔出         | KB_REMOVED  | 释放 HID 接管和输入缓冲，记录 USB 审计日志（拔出）    |
| KB_MAPPED   | USB 拔出         | KB_REMOVED  | 清理 OTG 映射，记录 USB 审计日志（拔出）         |
| KB_REJECTED | USB 拔出         | KB_REMOVED  | 记录 USB 审计日志（拔出）                   |


**说明：**

- KB_WAITING 状态下，键盘的 HID 输入由装置侧捕获，不透传到受控主机
- 1234 必须是顺序输入，中途错误需清空当前输入序列，并继续等待下一次完整输入
- 键盘验证不设置固定等待超时；键盘拔出时结束本次验证流程，释放 HID 接管状态和临时输入缓冲
- KB_REJECTED 状态下，键盘不映射到受控主机，等待 USB 拔出后清理资源

## 3. 鼠标设备状态机

```
        ┌────────────────┐
        │  USB 插入事件   │
        └───────┬────────┘
                ↓
        ┌────────────────┐
        │ MOUSE_DETECTED │ 已检测为鼠标
        └───────┬────────┘
                ↓ 自动映射
        ┌───────┴───────┐
        │               │
    映射成功          映射失败
        ↓               ↓
┌──────────────┐ ┌────────────────┐
│ MOUSE_MAPPED │ │ MOUSE_REJECTED │
│ 鼠标已映射     │ │ 映射失败        │
└──────┬───────┘ └───────┬────────┘
       │                 │
       ↓ USB 拔出         ↓ USB 拔出
┌───────────────┐ ┌───────────────┐
│ MOUSE_REMOVED │ │ MOUSE_REMOVED │
└───────────────┘ └───────────────┘
```

**状态转换表：**


| 当前状态           | 事件       | 下一状态           | 动作                        |
| -------------- | -------- | -------------- | ------------------------- |
| MOUSE_DETECTED | OTG 映射成功 | MOUSE_MAPPED   | 记录 USB 审计日志（鼠标映射成功）       |
| MOUSE_DETECTED | OTG 映射失败 | MOUSE_REJECTED | 记录 USB 审计日志（鼠标映射失败）       |
| MOUSE_DETECTED | USB 拔出   | MOUSE_REMOVED  | 记录 USB 审计日志（拔出）           |
| MOUSE_MAPPED   | USB 拔出   | MOUSE_REMOVED  | 清理 OTG 映射，记录 USB 审计日志（拔出） |
| MOUSE_REJECTED | USB 拔出   | MOUSE_REMOVED  | 记录 USB 审计日志（拔出）           |


**说明：**

- 鼠标无安全注入风险，插入后自动映射，零用户干预
- 鼠标不需要白名单、不需要病毒扫描、不需要验证

## 4. 管理端连接状态机

```
        ┌──────────────────┐
        │  初始状态（未连接） │
        └────────┬─────────┘
                 ↓ 用户点击登录
        ┌──────────────────┐
        │   CONNECTING     │ 连接中（TCP/TLS）
        └────────┬─────────┘
                 │
        ┌────────┴────────┐
        │ 连接失败         │ 连接成功
        ↓                 ↓
    DISCONNECTED  ┌──────────────────┐
    未连接         │  AUTHENTICATING  │ 装置侧账号鉴权
    （提示错误）    └────────┬─────────┘
                           │
                  ┌────────┴────────┐
                  │ 鉴权失败         │ 鉴权成功
                  ↓                 ↓
             AUTHENTICATING  ┌──────────────────┐
             保持连接         │  CHECK_LICENSE   │ 检查授权状态
             提示密码错误      └────────┬─────────┘
                                      │
                        ┌─────────────┼─────────────┐
                        │ 未授权       │ 已授权       │ 授权到期
                        ↓             ↓             ↓
                ┌──────────────┐ ┌────────────┐ ┌──────────────┐
                │AUTH_REQUIRED │ │LOADING_    │ │LICENSE_      │
                │独立授权页      │ │CONFIG      │ │EXPIRED       │
                └──────┬───────┘ │拉取配置中    │ │授权到期停用    │
                       │         └─────┬──────┘ └──────┬───────┘
                授权成功 → 跳登录页       │               │
                清空临时会话        ┌────┴────┐    用户关闭管理端
                              拉取成功    拉取失败   网络异常
                                 ↓          │         │
                           ┌──────────┐     └────┬────┘
                           │CONNECTED │          ↓
                           │已连接     │    ┌──────────────┐
                           └────┬─────┘    │ DISCONNECTED │
                                │          │ 未连接        │
                   ┌────────────┼─────┐    └──────────────┘
                   │ 网络异常/超时│登出  │
                   ↓            ↓
              ┌──────────────┐ ┌──────────────┐
              │ DISCONNECTED │ │ DISCONNECTED │
              │ 连接断开      │ │ 已登出        │
              │ 保留当前页面   │ │ 跳登录页      │
              │ 进程内状态     │└──────────────┘
              └──────────────┘
```

装置未授权时，任一有效账号鉴权通过后均进入 `AUTH_REQUIRED`。独立授权页属于进入主系统前的装置激活流程，不按登录后三权权限限制；授权成功后返回登录页并清空当前临时会话，用户需重新登录。授权失败时（文件格式错误、校验不通过等），保持 `AUTH_REQUIRED` 状态，提示失败原因，用户可重新上传授权文件。

授权到期时，装置侧在 CHECK_LICENSE 阶段返回授权到期状态，管理端进入 `LICENSE_EXPIRED`，提示"授权已到期"，系统功能停用。用户关闭管理端或网络异常时进入 `DISCONNECTED`。

配置拉取失败时（如网络超时或装置端返回错误），管理端从 `LOADING_CONFIG` 进入 `DISCONNECTED`，清空进程内会话状态，提示"配置拉取失败，请重新连接"并返回登录页。

**全局状态字段（Pinia store）：**


| 字段                 | 类型     | 说明                                                                                                                        |
| ------------------ | ------ | ------------------------------------------------------------------------------------------------------------------------- |
| `connectionStatus` | enum   | DISCONNECTED / CONNECTING / AUTHENTICATING / CHECK_LICENSE / AUTH_REQUIRED / LICENSE_EXPIRED / LOADING_CONFIG / CONNECTED |
| `deviceIP`         | string | 当前进程内输入并连接的装置 IP，不持久化                                                                                                     |
| `currentUser`      | object | `{username, role}`                                                                                                        |
| `sessionToken`     | string | 装置端返回的临时会话 token，仅进程内保存                                                                                                   |
| `deviceConfig`     | object | C04 装配的当前页面状态，仅进程内保存，管理端与装置断开连接后保留用于展示                                                                                    |


**管理端与装置断开连接行为：**

- 管理端与装置意外断开连接不自动跳登录页，不清空当前页面展示内容。
- 任何需要下发到装置侧的动作在点击时失败提示。
- 管理端关闭、崩溃或用户主动登出后，进程内状态清空，重新打开必须重新输入 IP、用户名和密码。

## 5. 病毒扫描任务状态机

每个 U 盘扫描为一个独立任务，由 S03 管理。装置同一时间只执行一个扫描任务，多个 U 盘同时插入时按 FIFO 顺序排队。

```
    提交扫描任务
        ↓
    ┌────────────┐
    │  PENDING   │ 等待执行（排队中）
    └─────┬──────┘
          │
     ┌────┴────────────┐
     │ USB 拔出         │ 获得执行槽位
     ↓                 ↓
┌──────────┐     ┌────────────┐
│CANCELLED │     │  RUNNING   │ 扫描中
│ 已取消    │     └─────┬──────┘
└──────────┘           │
               ┌───────┼───────────┐
               │ 扫描完成│           │ clamd 异常/
               │       │ USB 拔出   │ 连接断开
               │       ↓           ↓
               │  ┌──────────┐ ┌──────────┐
               │  │CANCELLED │ │  FAILED  │
               │  │ 已取消    │ │  失败     │
               │  └──────────┘ └────┬─────┘
               │                    │ USB 拔出
               │                    ↓
               │               ┌──────────┐
               │               │CANCELLED │
               │               │ 已取消    │
               │               └──────────┘
          ┌────┴────┐
          │         │
        未发现    发现病毒
          ↓         ↓
    ┌─────────┐ ┌──────────┐
    │  CLEAN  │ │ INFECTED │
    │  干净    │ │ 发现病毒  │
    └─────────┘ └──────────┘
```

**状态转换表：**


| 当前状态    | 事件            | 下一状态      | 动作                              |
| ------- | ------------- | --------- | ------------------------------- |
| PENDING | 获得执行槽位        | RUNNING   | 调用 clamd 开始全盘扫描                 |
| PENDING | USB 拔出        | CANCELLED | 移出队列，记录 USB 审计日志                |
| RUNNING | 扫描完成：未发现病毒    | CLEAN     | 通知 U 盘状态机进入 CLEAN               |
| RUNNING | 扫描完成：发现病毒     | INFECTED  | 输出病毒文件路径列表，通知 U 盘状态机进入 INFECTED |
| RUNNING | USB 拔出        | CANCELLED | 中断扫描，释放 clamd 连接                |
| RUNNING | clamd 异常/连接断开 | FAILED    | 通知 U 盘状态机进入 SCAN_FAILED         |
| FAILED  | USB 拔出        | CANCELLED | 释放扫描任务资源                        |


**调度策略：**

- 同一时间只有一个扫描任务处于 RUNNING 状态，避免嵌入式环境（4GB 内存）资源争用
- 多个 U 盘同时插入时，按插入时间 FIFO 排队
- PENDING 状态的 U 盘在管理端不可见为"已映射"，设备处于等待扫描状态

## 6. 系统程序升级状态机

```
    初始状态（正常运行）
        ↓ 收到升级包
    ┌──────────────┐
    │  VALIDATING  │ 校验中
    └──────┬───────┘
           │
    ┌──────┴──────────┐
    │ 校验失败         │ 校验成功
    ↓                 ↓
  IDLE            ┌────────────┐
  恢复正常         │  UPGRADING │ 升级中（替换程序）
                  └─────┬──────┘
                        │
                 ┌──────┴──────┐
                 │ 升级成功     │ 升级失败
                 ↓             ↓
            ┌────────────┐ ┌────────────┐
            │ RESTARTING │ │  ROLLBACK  │
            │ 重启服务    │ │ 回滚原程序  │
            └─────┬──────┘ └─────┬──────┘
                  ↓               ↓
            IDLE（新版本）    IDLE（原版本）
```

**校验规则：**

- 文件格式和完整性（SHA256 校验和）
- 升级包版本必须大于当前系统版本，不允许降版本

## 7. 病毒库升级状态机

病毒库升级与系统程序升级的核心差异：升级后不重启装置服务，不断开管理端连接。

```
    初始状态（正常运行）
        ↓ 收到 .zip 病毒库升级包
    ┌────────────────┐
    │  VDB_VALIDATING│ 校验中
    └───────┬────────┘
            │
    ┌───────┴─────────┐
    │ 校验失败         │ 校验成功
    ↓                 ↓
  IDLE            ┌────────────────┐
  恢复正常         │ VDB_REPLACING  │ 替换病毒库文件
                  └───────┬────────┘
                          │
                   ┌──────┴──────┐
                   │ 替换成功     │ 替换失败
                   ↓             ↓
            ┌────────────┐ ┌──────────────┐
            │VDB_RELOAD  │ │ VDB_ROLLBACK │
            │通知 clamd   │ │ 回滚旧病毒库  │
            │重新加载     │ └──────┬───────┘
            └─────┬──────┘        ↓
                  ↓           IDLE（旧版本）
            IDLE（新版本）
```

**校验规则：**

- 文件格式（`.zip`）
- 病毒库版本必须高于当前版本，不允许降版本
- 版本号从 `V3.0.0.0` 开始递增，递增时逢数字 `4` 跳过（如 `V3.0.0.3` → `V3.0.0.5`）

**与系统程序升级的区别：**


| 维度       | 系统程序升级         | 病毒库升级                     |
| -------- | -------------- | ------------------------- |
| 文件格式     | `.bin`         | `.zip`                    |
| 升级后行为    | 重启装置服务，管理端连接断开 | 通知 clamd 重新加载，不重启，不断连     |
| 对已映射设备影响 | 连接断开，需重新连接     | 已映射 U 盘不受影响，新插入 U 盘使用新病毒库 |
