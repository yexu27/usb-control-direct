# 统一错误码表

装置端通过 RspCommon.error_code 或特定响应消息的 error_code 字段返回错误码。错误码为字符串枚举，管理端根据错误码决定提示方式。

---

## 1. 全局错误码

全局错误码适用于所有消息类型，任何接口均可能返回。装置端执行业务逻辑前先检查全局条件（会话、授权、权限），全局错误码优先级高于模块错误码。


| 错误码                  | 含义           | 管理端提示文案                       |
| -------------------- | ------------ | ----------------------------- |
| UNAUTHENTICATED      | 会话无效或已过期     | 清空进程内会话状态，跳转登录页（无弹窗提示）        |
| UNAUTHORIZED         | 装置未授权        | "装置未授权，请先完成授权"                |
| PERMISSION_DENIED    | 当前角色无权限      | "当前用户无权执行此操作"                 |
| ACCOUNT_LOCKED       | 账号已锁定        | "用户已被锁定，请5分钟后重试"              |
| VALIDATION_FAILED    | 请求参数校验失败     | 展示 error_message 原文           |
| DEVICE_BUSY          | 装置正在执行其他操作   | "装置正在处理其他操作，请稍后重试"            |
| INTERNAL_ERROR       | 装置内部错误       | "装置内部错误，请联系管理员"               |


---

## 2. 登录鉴权错误码（0x0001 - 0x0008）

登录错误码通过 RspLogin.error_code（0x0002）返回。授权文件上传错误码通过 RspUploadLicense.error_code（0x0008）返回。


| 错误码                      | 适用消息码  | 含义             | 管理端提示文案                        |
| ------------------------ | ------ | -------------- | ------------------------------ |
| USER_OR_PASSWORD_ERROR   | 0x0001 | 用户名或密码错误       | "用户名或密码错误"                     |
| DEVICE_UNAUTHORIZED      | 0x0001 | 装置未授权，需进入授权页    | 跳转独立授权页                        |
| LICENSE_FORMAT_ERROR     | 0x0007 | 授权文件格式错误       | "授权文件格式错误，请检查文件"               |
| LICENSE_VERIFY_FAILED    | 0x0007 | 授权文件校验失败（签名不匹配、与装置不匹配或已过有效期） | "授权文件校验失败，请确认文件与本装置匹配"  |
| LICENSE_EXPIRED          | 0x0007 | 上传的授权文件已过有效期   | "授权文件已过有效期，请重新获取授权文件"          |

说明：

- RspLogin.error_code 可返回全局错误码 ACCOUNT_LOCKED（连续 5 次登录失败锁定）。
- CMD_AUTH_STATUS_QUERY（0x0003）的授权状态通过 RspAuthStatus.auth_status 字段返回（authorized / unauthorized / expired / failed），不使用错误码机制。管理端根据 auth_status 值判断授权状态并展示对应提示。

## 9. 错误码使用规则

- 装置端在 RspCommon.error_code 或特定响应的 error_code 字段中返回上述字符串值。
- 管理端优先按 error_code 匹配提示文案；未匹配到已知错误码时，直接展示 error_message 原文。
- 全局错误码（第 1 节）优先级高于模块错误码：装置端先检查会话、授权、权限等全局条件，通过后再执行业务逻辑。
- UNAUTHENTICATED 不弹窗提示，直接清空会话跳转登录页。
- error_message 由装置端填写中文文本，作为兜底提示。管理端在已知错误码场景下使用本表定义的提示文案，不依赖 error_message。
- 同一错误码可出现在多个模块章节中（如 FORMAT_ERROR 同时出现在策略导入和系统管理），含义按模块上下文区分，error_message 提供具体描述。

### 查询类命令错误处理约定

查询类命令（CMD_LIST_WHITELIST、CMD_GET_CONNECTED_DEVICES、CMD_GET_FILE_POLICY、CMD_QUERY_LOGS、CMD_GET_SYSTEM_INFO、CMD_LIST_USERS、CMD_AUTH_STATUS_QUERY、CMD_GET_MACHINE_CODE）的错误处理：

- 全局错误（会话无效、未授权、无权限等）：装置端返回 RspCommon（0xFF00）携带 error_code，管理端通过消息类型码区分专用响应和通用错误响应。
- 业务正常但无数据：返回专用响应，列表字段为空数组。
- 装置内部错误（如数据库查询失败）：装置端返回 RspCommon 携带 INTERNAL_ERROR。
