# 00 PRD 与原型图对齐分析

## 说明

- 需求判断以 PRD v1.0.1 为唯一准则
- 原型图（USB安全管理系统原型图-v1.0.1.html）仅作为页面布局、交互方式、视觉层级和用户操作路径参考
- PRD 与原型冲突时，以 PRD 为准

---

## 1. PRD 定义的核心功能

本节按 `03-modules.md` 的 Fxx 需求功能模块口径整理。Fxx 表示 PRD 定义的业务能力，不等同于程序实现模块；程序实现模块在架构文档中使用 `Sxx/Cxx` 表示。

### 1.1 管理端页面型需求功能


| 编号  | 功能模块   | 说明                                                                                    |
| --- | ------ | ------------------------------------------------------------------------------------- |
| F01 | 登录与授权  | 登录页输入装置 IP、用户名和密码，由装置侧完成账号鉴权、授权状态校验和角色返回；未授权时任一有效账号可进入独立授权页，授权成功后返回登录页重新登录            |
| F02 | 文件访问控制 | 配置可执行程序访问控制、介质自动读取控制、文件类型访问控制；可执行程序访问控制内置 `dll`、`exe`、`PE`、`ELF` 4 项，文件类型黑名单出厂预置 38 项 |
| F03 | U盘设备控制 | 管理普通 U 盘白名单，支持装置端添加、管理端添加、修改权限/描述、删除；序列号唯一，权限为只读/读写                                   |
| F04 | 策略管理   | 导出当前 U 盘设备控制和文件访问控制策略为 `.bin` 文件，导入 `.bin` 策略文件并整体覆盖；导入失败原策略不变                        |
| F05 | 日志管理   | 查询、导出、清理 USB 审计日志、恶意代码检测日志、操作日志；三类日志均存储在装置侧，管理端不持久化日志                                 |
| F06 | 系统管理   | 系统升级、病毒库升级、授权信息管理、自定义设备描述；系统升级包为 `.bin`，病毒库升级包为 `.zip`，病毒库版本递增逢数字 4 跳过                |
| F07 | 用户管理   | 查看用户列表，新建用户，删除非内置用户，重置密码；三个内置用户不可删除，已删除用户名不可复用                                        |


### 1.2 装置运行型需求功能


| 编号  | 功能模块        | 说明                                                                                      |
| --- | ----------- | --------------------------------------------------------------------------------------- |
| F08 | USB 设备准入与映射 | 对存储设备、键盘、鼠标、复合设备执行准入控制和 OTG 映射；未认证 U 盘禁止映射，键盘需顺序输入 `1234` 验证且验证过程不透传，鼠标自动映射，复合设备按接口分别处理 |
| F09 | 恶意代码检测      | 已认证 U 盘映射前由 ClamAV 全盘扫描，命中文件添加 `[病毒禁止访问]` 前缀并阻断访问；病毒库不可用或扫描失败时拒绝映射，检测日志存储在装置侧           |

## 3. PRD 与原型一致的部分

- 登录页字段：装置IP / 用户名 / 密码 / 登录按钮
- 授权页两个入口：下载机器码 / 上传授权文件
- 菜单结构与角色权限对应关系
- 白名单展示字段（序列号 / 描述 / 权限 / 添加方式 / 添加时间）
- 文件访问控制三个开关 + 黑名单表格分页
- 策略导入导出两张卡片布局
- 日志管理 Tab 结构和查询/导出/清理操作入口
- 系统管理四个功能块的布局
- 用户管理列表结构

## 4. PRD 有但原型缺失的部分（原型缺失）


| 功能             | PRD 规则                       | 影响                 |
| -------------- | ---------------------------- | ------------------ |
| 管理端与装置断开连接提示   | 连接断开后保留当前页面进程内状态，点击下发动作时提示失败 | 需要全局连接状态和动作失败提示    |
| 用户状态"锁定"       | 5次登录失败后账号锁定，用户表需展示锁定状态       | 用户管理页需锁定 badge     |
| 操作日志"操作日志类型"筛选 | 操作日志支持按类型精确筛选                | 操作日志 Tab 需增加类型下拉筛选 |
| 白名单权限修改入口      | PRD 明确可设置只读/读写权限             | 白名单行需要权限修改操作       |
| 日志清理时间范围输入     | 清理时需选择日志类型和时间范围              | 清理弹窗需增加时间范围选择      |
| 恶意代码检测日志"病毒"列  | 展示字段包含"病毒（病毒名称或恶意代码信息）"      | 表格需增加病毒列           |
| 操作日志"操作日志类型"列  | 展示字段包含操作日志类型                 | 表格需增加操作日志类型列       |
| 登录失败提示文案       | 5次失败提示"用户已被锁定，请5分钟后重试"       | 登录页错误提示逻辑          |

## 5. 原型有但 PRD 未定义的部分


| 功能          | 说明                                     | 处理            |
| ----------- | -------------------------------------- | ------------- |
| 安全U盘自由使用复选框 | 原型 U盘设备控制页有此开关，PRD明确本版本不支持，原型已标注"延后实现" | 本版本不实现，UI 不展示 |
| 安全U盘自动升级按钮  | 系统管理页有该按钮，PRD明确本版本不支持                  | 本版本不实现，UI 不展示 |

## 6. PRD 与原型冲突的部分


| 冲突点        | PRD                    | 原型                        | 结论                 |
| ---------- | ---------------------- | ------------------------- | ------------------ |
| 恶意代码检测日志字段 | 需展示"病毒"列（病毒名称）         | 原型无该列                     | 以 PRD 为准，补充病毒列     |
| 操作日志字段     | 需展示"操作日志类型"列           | 原型只有时间/用户/内容三列            | 以 PRD 为准，补充操作日志类型列 |
| 登录页装置 IP   | 输入框为空，不预填、不记住、不展示默认 IP | 原型登录页示例值为 `192.168.1.100` | 以 PRD 为准，输入框保持空    |

## 7. 对齐结论

当前对齐结论已覆盖本版本架构设计。架构文档以 PRD 为需求基准，原型与 PRD 不一致时以 PRD 为准。
